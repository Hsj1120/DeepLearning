# 1. 이번 강의에서 배우는 것

이번부터는
**비지도 딥러닝**을 실제로 구현한다.

그리고 첫 번째 모델은:

**Self-Organizing Map, SOM**

이다.

------

# 2. 이번에 해결할 문제

이번에는 SOM을 사용해서
**사기 탐지 Fraud Detection** 문제를 해결한다.

상황은 이렇다.

우리가 은행에서 일하는 딥러닝 과학자라고 하자.

은행 고객들이 고급 신용카드를 신청했다.

그리고 우리에게는
그 신청서에 적힌 고객 정보 데이터가 주어졌다.

목표는:

```text
잠재적으로 사기를 친 고객 찾기
```

이다.

------

# 3. 지도학습으로 하지 않는다

여기서 중요한 점이 있다.

우리는 고객마다:

```text
사기다 / 사기가 아니다
```

를 예측하는 지도학습 모델을 만들지 않는다.

즉:

```text
X → y 예측
```

이런 방식이 아니다.

------

# 4. 비지도 학습으로 한다

이번에는 **비지도 딥러닝**을 사용한다.

비지도 학습은 정답 label을 직접 사용하지 않는다.

대신 데이터 안에서:

- 패턴
- 이상한 점
- 비슷한 고객 그룹
- 일반적인 고객과 다른 고객

을 찾는다.

그리고 그중 하나의 패턴이
**잠재적 사기 고객**일 수 있다.

------

# 5. 사용할 데이터셋

사용할 데이터셋은:

**Credit Card Applications.csv**

이다.

이 데이터셋은
UCI Machine Learning Repository의
**Statlog Australian Credit Approval Data Set**이다.

------

# 6. 데이터셋 구조

이 데이터셋에서:

```text
행(row) = 고객 1명
열(column) = 고객의 정보
```

이다.

예를 들어 한 줄은
한 명의 고객 신청서를 의미한다.

------

# 7. 컬럼 이름이 이상한 이유

데이터셋의 attribute 이름과 값은
기밀 보호를 위해 의미 없는 기호로 바뀌어 있다.

즉, 우리가 봐도:

```text
이 컬럼이 소득인지
이 컬럼이 나이인지
이 컬럼이 직업인지
```

정확히 알 수 없다.

그래서 사람이 직접 보고
사기 고객을 찾기 어렵다.

👉 그래서 모델이 필요하다.

------

# 8. SOM이 여기서 하는 일

SOM은 고객들을 보고
비슷한 고객끼리 묶는다.

쉽게 말하면:

```text
고객 데이터
→ SOM
→ 비슷한 고객끼리 지도 위에 배치
```

한다.

이 과정은 일종의
**고객 세분화 Customer Segmentation**처럼 볼 수 있다.

------

# 9. 여기서 Pattern이란?

강의에서 말하는 pattern은
복잡하게 생각할 필요 없다.

여기서는:

```text
비슷한 특징을 가진 고객 그룹
```

이라고 보면 된다.

예를 들어:

- 일반적인 신청자 그룹
- 승인받은 고객 그룹
- 거절된 고객 그룹
- 다른 고객들과 많이 다른 이상한 고객 그룹

이런 식이다.

------

# 10. 사기 고객은 왜 Outlier일까?

사기는 보통 일반적인 규칙에서 벗어난다.

즉, 대부분의 정상 고객들과
다른 특징을 가질 가능성이 크다.

그래서 사기 고객은 SOM에서:

**Outlier**

즉:

**이상치**

처럼 나타날 수 있다.

------

# 11. SOM에서 사기 고객을 찾는 방식

흐름은 이렇다.

```text
고객 데이터 입력
→ SOM이 고객들을 2차원 지도에 배치
→ 일반 고객과 멀리 떨어진 node 찾기
→ 그 node에 매칭된 고객 찾기
→ 잠재적 사기 고객 리스트 반환
```

------

# 12. 입력 데이터 X 만들기

먼저 데이터셋에서
입력 데이터 `X`를 만든다.

```python
X = dataset.iloc[:, :-1].values
```

뜻은:

```text
모든 행 선택
마지막 컬럼 제외
나머지 모든 컬럼 선택
```

이다.

즉, 고객 ID부터 attribute 14까지를
`X`에 넣는다.

------

# 13. 결과 데이터 y 만들기

그리고 마지막 컬럼은 `y`로 따로 뺀다.

```python
y = dataset.iloc[:, -1].values
```

마지막 컬럼은
신용카드 신청 승인 여부이다.

```text
0 = 승인 안 됨
1 = 승인됨
```

------

# 14. 그런데 왜 y를 만들까?

여기서 헷갈리면 안 된다.

`y`를 만들었다고 해서
지도학습을 하는 것은 아니다.

SOM 학습에는 `y`를 사용하지 않는다.

SOM 학습에는 오직:

```text
X만 사용
```

한다.

------

# 15. y는 나중에 해석용이다

`y`는 나중에 결과를 해석할 때 사용한다.

예를 들어 사기 의심 고객 중에서:

```text
승인받은 고객인가?
승인 거절된 고객인가?
```

를 구분하기 위해 필요하다.

특히 더 중요한 것은:

```text
사기 의심 + 카드 승인됨
```

인 고객이다.

왜냐하면 이미 승인된 사기 고객이면
은행 입장에서 더 위험하기 때문이다.

------

# 16. SOM의 neuron 구조

SOM의 각 neuron은 weight vector를 가진다.

이 weight vector의 크기는
입력 고객 데이터의 크기와 같다.

강의에서는 고객 데이터가:

```text
고객 ID + 14개 attribute
= 총 15개 값
```

이라고 설명한다.

그래서 각 neuron도:

```text
15개의 weight
```

를 가진다.

------

# 17. Winning Node

각 고객이 들어오면
SOM은 그 고객과 가장 비슷한 neuron을 찾는다.

이 neuron을:

**Winning Node**

라고 한다.

이전 강의에서 말한:

**BMU, Best Matching Unit**

과 같은 개념이다.

즉:

```text
현재 고객과 가장 가까운 neuron
```

이다.

------

# 18. 학습 과정

고객 한 명이 들어오면:

```text
1. 고객과 가장 가까운 neuron 찾기
2. 그 neuron을 winning node로 선택
3. winning node 주변 neuron들도 같이 업데이트
4. 고객 데이터 쪽으로 조금 이동
```

한다.

이 과정을 모든 고객에 대해 반복한다.

그리고 여러 번 반복하면서
SOM이 점점 정리된다.

------

# 19. 최종 결과

반복 학습이 끝나면
2차원 SOM 지도가 만들어진다.

이 지도에서 중요한 것은:

```text
일반적인 고객들과 멀리 떨어진 neuron
```

이다.

이런 neuron이 outlier일 가능성이 있다.

그리고 이 outlier neuron에 연결된 고객들이
잠재적 사기 고객 후보가 된다.

------

# 20. MID란?

Outlier neuron을 찾기 위해 사용하는 값이 있다.

그게:

**MID**

이다.

MID는:

**Mean Interneuron Distance**

의 약자이다.

뜻은:

```text
한 neuron과 주변 neuron들 사이의 평균 거리
```

이다.

------

# 21. MID가 큰 neuron의 의미

어떤 neuron의 MID가 크다는 것은:

```text
주변 neuron들과 많이 다르다
```

는 뜻이다.

즉, 그 neuron은 주변과 떨어져 있다.

그래서 outlier일 가능성이 높다.

------

# 22. Outlier와 Fraud 연결

사기 고객은 일반 고객과 다른 패턴을 가질 가능성이 크다.

그래서 SOM에서 outlier neuron을 찾으면
그 neuron에 매칭된 고객들을
사기 의심 고객으로 볼 수 있다.

------

# 23. Feature Scaling이 필요한 이유

SOM을 학습하기 전에
feature scaling을 해야 한다.

이유는 SOM이 거리 계산을 하기 때문이다.

값의 범위가 다르면
거리 계산이 한쪽 변수에 치우칠 수 있다.

그래서 모든 값을 0과 1 사이로 맞춘다.

------

# 24. MinMaxScaler 사용

코드는 이렇게 쓴다.

```python
from sklearn.preprocessing import MinMaxScaler

sc = MinMaxScaler(feature_range = (0, 1))
X = sc.fit_transform(X)
```

뜻은:

```text
X의 최소값과 최대값을 기준으로
모든 값을 0~1 사이로 변환
```

하는 것이다.

------

# 25. 전체 흐름 정리

이번 강의 흐름은 이렇다.

```text
데이터셋 불러오기
→ X와 y 분리
→ X만 SOM 학습에 사용
→ y는 나중에 승인 여부 확인용으로 사용
→ X를 0~1 사이로 정규화
→ 다음 강의에서 SOM 학습 진행
```

------

# 26. 한 줄 핵심 정리

👉 이번 실습은
**신용카드 신청 고객 데이터를 SOM에 넣고,
일반 고객과 다른 outlier neuron을 찾아
잠재적 사기 고객을 탐지하는 비지도 딥러닝 예제**이다.
