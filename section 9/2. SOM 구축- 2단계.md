# 1. 이번 강의에서 배우는 것

이제 이번 강의에서는 **SOM을 실제로 학습시키는 과정**을 진행한다.

------

# 2. SOM을 구현하는 방법

SOM을 만드는 방법은 두 가지가 있다.

1. 직접 처음부터 구현하기
2. 다른 개발자가 만든 SOM 클래스를 사용하기

이번 강의에서는 직접 구현하지 않고 이미 만들어진 코드를 사용한다.

------

# 3. 왜 직접 구현하지 않을까?

SOM은 scikit-learn에 기본 구현이 없다.

그래서 다른 개발자가 만든 구현체를 가져와야 한다.

이번에 사용할 구현체는 **MiniSom 1.0**이다.

MiniSom은 NumPy 기반의 Self-Organizing Map 구현체이다.

------

# 4. MiniSom을 사용하는 이유

MiniSom을 사용하는 이유는 두 가지이다.

첫째, 구현이 잘 되어 있다.

둘째, 이번 파트에서는 너무 어렵게 시작하지 않기 위해서이다.

나중에 배울 Boltzmann Machine이나 Stacked Autoencoder는 직접 클래스를 만들면서 더 깊게 구현할 예정이다.

그래서 이번 SOM에서는 이미 만들어진 클래스를 사용해서 개념과 흐름에 집중한다.

------

# 5. MiniSom 파일 준비

MiniSom을 사용하려면 작업 폴더 안에 다음 파일이 있어야 한다.

```python
MiniSom.py
```

즉, 데이터셋이 있는 폴더와 같은 위치에 `MiniSom.py` 파일이 있어야 한다.

그래야 Python 코드에서 MiniSom 클래스를 불러올 수 있다.

------

# 6. MiniSom import

코드는 이렇게 쓴다.

```python
from minisom import MiniSom
```

또는 강의 파일 구조에 따라:

```python
from MiniSom import MiniSom
```

처럼 쓸 수 있다.

뜻은:

```text
MiniSom.py 파일에서
MiniSom 클래스를 가져온다
```

이다.

------

# 7. SOM 객체 만들기

이제 SOM 객체를 만든다.

```python
som = MiniSom(x = 10, y = 10, input_len = 15, sigma = 1.0, learning_rate = 0.5)
```

여기서 `som`은 앞으로 학습시킬 Self-Organizing Map 객체이다.

------

# 8. x = 10, y = 10 의미

```python
x = 10
y = 10
```

은 SOM 지도의 크기를 의미한다.

즉:

```text
10 × 10 SOM map
```

을 만든다는 뜻이다.

그래서 총 neuron 개수는:

```text
100개
```

이다.

------

# 9. 왜 10 × 10으로 할까?

이 값은 어느 정도 임의로 정할 수 있다.

너무 작으면 outlier를 찾기 어렵다.

너무 크면 데이터에 비해 map이 과하게 커질 수 있다.

이번 데이터셋은 고객 수가 아주 많지 않기 때문에 10 × 10 정도로 설정한다.

------

# 10. input_len = 15 의미

```python
input_len = 15
```

는 입력 feature 개수를 의미한다.

이번 `X`에는:

```text
CustomerID + 14개 attribute
```

가 들어 있다. 그래서 총 15개 값이다.

즉, SOM의 각 neuron은 15개의 weight를 가진다.

------

# 11. CustomerID도 넣는 이유

사실 CustomerID는 패턴 분석에는 큰 의미가 없다.

하지만 나중에 사기 의심 고객을 찾았을 때 그 고객이 누구인지 확인해야 한다.

그래서 CustomerID도 `X`에 포함시킨다.

------

# 12. sigma = 1.0 의미

```python
sigma = 1.0
```

은 neighborhood radius와 관련 있다.

쉽게 말하면:

```text
BMU 주변에서 같이 업데이트될 범위
```

라고 보면 된다.

이번에는 기본값인 1.0을 사용한다.

------

# 13. learning_rate = 0.5 의미

```python
learning_rate = 0.5
```

는 weight를 얼마나 크게 업데이트할지 정하는 값이다.

쉽게 말하면:

```text
입력 데이터 쪽으로 얼마나 빠르게 이동할 것인가
```

이다.

learning rate가 크면 빠르게 수렴하고, 작으면 천천히 학습한다.

------

# 14. Weight 초기화

SOM을 학습하기 전에 weight를 먼저 초기화해야 한다.

코드는 이렇게 쓴다.

```python
som.random_weights_init(X)
```

뜻은:

```text
SOM neuron들의 weight를 랜덤하게 초기화한다
```

이다.

처음 weight는 보통 0에 가까운 작은 값으로 설정된다.

------

# 15. 왜 weight 초기화가 필요할까?

SOM의 각 neuron은 입력 데이터와 비교하기 위한 weight vector를 가진다.

그런데 처음에는 아직 학습 전이기 때문에 적절한 위치를 모른다.

그래서 먼저 랜덤한 위치에서 시작하게 만든다.

그다음 학습하면서 입력 데이터 구조에 맞게 weight가 이동한다.

------

# 16. SOM 학습시키기

이제 SOM을 학습시킨다.

```python
som.train_random(data = X, num_iteration = 100)
```

뜻은:

```text
X 데이터를 사용해서
SOM을 100번 반복 학습시킨다
```

이다.

------

# 17. train_random이 하는 일

`train_random()`은 내부적으로 SOM 학습 과정을 수행한다.

즉:

```text
입력 row 선택
→ BMU 찾기
→ BMU와 주변 neuron 업데이트
→ 반복
```

을 수행한다.

여기서는 이 과정을 100번 반복한다.

------

# 18. 왜 X만 사용하고 y는 사용하지 않을까?

중요한 점은 이것이다.

SOM은 비지도 학습이다.

그래서 학습할 때:

```text
X만 사용
```

한다.

`y`는 사용하지 않는다.

즉:

```text
som.train_random(data = X, ...)
```

이지,

```text
som.train_random(data = X, y)
```

가 아니다.

------

# 19. y는 어디에 쓰일까?

`y`는 나중에 결과를 해석할 때 사용한다.

예를 들어 SOM이 사기 의심 고객을 찾았을 때:

```text
이 고객이 카드 승인을 받았는지
거절당했는지
```

를 확인하는 용도이다.

------

# 20. 전체 코드 흐름

이번 강의의 핵심 코드는 이렇게 정리할 수 있다.

```python
from minisom import MiniSom

som = MiniSom(x = 10, y = 10, input_len = 15, sigma = 1.0, learning_rate = 0.5)

som.random_weights_init(X)

som.train_random(data = X, num_iteration = 100)
```

------

# 21. 전체 흐름 정리

이번 강의 흐름은 이렇다.

```text
MiniSom 클래스 import
→ SOM 객체 생성
→ 10 × 10 map 설정
→ input_len = 15 설정
→ sigma와 learning_rate 설정
→ weight 랜덤 초기화
→ X 데이터로 SOM 학습
→ 다음 강의에서 결과 시각화
```

------
