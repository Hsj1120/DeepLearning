# 1. 이번 강의에서 하는 것

이번 강의는
**Hybrid Deep Learning Model 만들기 Part 1**이다.

Part 1의 목표는:

```text
비지도 딥러닝 branch 만들기
```

이다.

여기서 사용할 모델은 **Self-Organizing Map, SOM**이다.

------

# 2. Part 1은 어렵지 않다

이번 Part 1은 이전에 했던 SOM 실습과 거의 같다.

즉:

```text
Credit Card Applications 데이터
→ SOM 학습
→ outlier neuron 찾기
→ fraud 후보 고객 찾기
```

를 다시 수행한다.

새로운 핵심은 Part 2에서 나온다.

------

# 3. Part 2에서 어려운 부분

Part 2에서는:

```text
비지도 학습 결과
→ 지도 학습 데이터로 변환
```

해야 한다.

즉, SOM 결과를 이용해서 ANN이 학습할 수 있는 형태로 만들어야 한다.

------

# 4. SOM 결과 확인

SOM을 실행하면 2차원 map이 나온다.

여기서 흰색에 가까운 neuron을 찾는다.

흰색 neuron은:

```text
MID가 큼
→ 주변 neuron들과 많이 다름
→ outlier
→ fraud 가능성 높음
```

이다.

------

# 5. Outlier neuron 선택

강의에서는 outlier neuron을 두 개 선택한다.

하나는 가장 흰색에 가까운 neuron이고, 다른 하나는 threshold를 조금 낮춰서 포함한 neuron이다.

즉:

```text
가장 강한 fraud 후보
+ 약간 덜 강하지만 의심되는 후보
```

를 함께 사용한다.

------

# 6. 선택 기준은 조금 임의적이다

여기서 중요한 점은 outlier neuron을 어디까지 선택할지는 어느 정도 분석자의 판단이다.

예를 들어:

```text
가장 흰 neuron만 선택
```

할 수도 있고,

```text
조금 덜 흰 neuron까지 포함
```

할 수도 있다.

------

# 7. 강의에서 선택한 좌표

강의에서는 두 outlier neuron의 좌표를 이렇게 잡는다.

```text
(5, 3)
(8, 3)
```

그래서 이 좌표에 매칭된 고객들을 fraud 후보로 가져온다.

------

# 8. Fraud 후보 추출

이전과 같이 `win_map()`을 사용한다.

흐름은 이렇다.

```text
mappings = som.win_map(X)
→ mappings[(5, 3)] 고객 추출
→ mappings[(8, 3)] 고객 추출
→ 두 리스트 합치기
→ inverse_transform으로 원래 값 복원
```

------

# 9. 결과

이렇게 해서 강의에서는 총 37명의 fraud 후보를 얻는다.

즉:

```text
SOM 기준으로 일반 고객과 다른 패턴을 가진 고객 37명
```

을 찾은 것이다.

------

# 10. 여기서 끝이 아니다

SOM만으로는 아직 부족하다.

SOM은 fraud 후보를 찾았지만, 각 고객에 대해:

```text
사기 확률이 몇 %인가?
```

를 알려주지는 않는다.

그래서 다음 단계가 필요하다.

------

# 11. 이제 지도학습으로 넘어가야 한다

ANN을 사용하려면 지도학습 데이터가 필요하다.

지도학습에는 반드시:

```text
X = 입력 데이터
y = 정답 데이터
```

가 있어야 한다.

그런데 원래 데이터에는 fraud 여부 정답이 없다.

------

# 12. 핵심 질문

이제 가장 중요한 질문은 이것이다.

```text
SOM으로 찾은 fraud 후보를 이용해서
ANN 학습용 y를 어떻게 만들 것인가?
```

즉:

```text
비지도 학습 결과
→ 지도학습의 dependent variable 생성
```

이 필요하다.

------

# 13. dependent variable이란?

dependent variable은 쉽게 말하면:

```text
모델이 맞춰야 하는 정답값
```

이다.

ANN에서는 이 값이 있어야 한다.

예를 들어:

```text
고객 A → fraud = 1
고객 B → fraud = 0
고객 C → fraud = 0
고객 D → fraud = 1
```

이런 값이 필요하다.

------

# 14. 다음 강의에서 할 일

다음 강의에서는 이 fraud 후보 리스트를 이용해서 ANN 학습용 dependent variable을 만든다.

즉:

```text
SOM으로 fraud 후보 찾기
→ fraud 후보는 1
→ 나머지는 0
→ ANN 학습용 y 생성
```

이 흐름으로 넘어간다.

------

# 15. 한 줄 핵심 정리

👉 Part 1은 **SOM으로 fraud 후보 고객을 다시 찾는 단계**이고, 진짜 핵심은 다음 Part 2에서 **이 SOM 결과를 이용해 ANN이 학습할 dependent variable을 만드는 것**이다.
