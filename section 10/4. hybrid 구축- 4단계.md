# 1. 이번 강의 목표

이번 강의의 목표는:

```text
SOM이 찾은 Fraud 후보 고객들의 실제 고객 ID를 추출하기
```

이다.

지금까지는 SOM이:

```text
이 neuron이 수상하다
```

까지만 알려줬다.

이제는:

```text
그래서 어떤 고객이 수상한데?
```

를 찾아야 한다.

------

# 2. 현재 상황

우리는 이미 SOM을 학습했고,

MID Map에서:

```text
흰색 neuron
```

을 찾았다.

흰색 neuron은:

```text
MID가 큼
→ 주변 neuron과 많이 다름
→ Outlier
→ Fraud 가능성 높음
```

이었다.

------

# 3. 문제점

하지만 현재 우리가 알고 있는 것은:

```text
Outlier neuron 위치
```

뿐이다.

예:

```text
(8,1)
(6,8)
```

------

# 4. 알고 싶은 것

우리가 진짜 원하는 것은:

```text
(8,1)에 어떤 고객이 들어있지?
(6,8)에 어떤 고객이 들어있지?
```

이다.

즉:

```text
Winning Node
→ Customer 목록
```

을 찾고 싶다.

------

# 5. 그런데 inverse mapping 함수가 없다

있으면 좋겠지만:

```text
Node 좌표
→ 고객 리스트
```

를 바로 반환하는 함수는 없다.

그래서 다른 방법을 사용한다.

------

# 6. win_map() 사용

MiniSom에는:

```python
som.win_map(X)
```

이라는 함수가 있다.

```python
mappings = som.win_map(X)
```

------

# 7. win_map()이 반환하는 것

반환값은:

```python
dictionary
```

이다.

구조는:

```text
Key
→ Winning Node 좌표

Value
→ 그 Node에 속한 고객들
```

이다.

------

# 8. 예시

예를 들어:

```python
mappings[(0,0)]
```

을 하면:

```text
(0,0) neuron에 속한 고객들
```

이 나온다.

------

# 9. dictionary 구조

대략 이런 느낌이다.

```python
{
 (0,0): [고객1, 고객2, 고객3],
 (0,1): [고객4],
 (0,2): [고객5, 고객6],
 ...
}
```

------

# 10. 왜 고객이 여러 명일까?

한 neuron은

```text
대표점
```

이다.

그래서 여러 고객이:

```text
가장 가까운 neuron
```

으로 같은 neuron을 선택할 수 있다.

즉:

```text
Neuron 하나
→ 여러 고객
```

이 가능하다.

------

# 11. 고객 데이터는 아직 Scaled 상태

현재 mappings 안의 고객 정보는:

```python
MinMaxScaler
```

를 적용한 값이다.

예:

```text
0.37
0.84
0.12
...
```

이런 값으로 보인다.

------

# 12. 왜 고객 ID가 이상하게 보일까?

CustomerID도 함께 Scaling 되었기 때문이다.

원래:

```text
15672829
```

였던 값이

```text
0.52
```

같이 바뀌어 있다.

------

# 13. Fraud 후보 Node 찾기

Map을 다시 본다.

강의에서는:

```text
(8,1)
(6,8)
```

두 개를 선택했다.

------

# 14. 첫 번째 Fraud Node 고객 가져오기

```python
mappings[(8,1)]
```

뜻:

```text
(8,1)에 속한 고객들
```

------

# 15. 두 번째 Fraud Node 고객 가져오기

```python
mappings[(6,8)]
```

뜻:

```text
(6,8)에 속한 고객들
```

------

# 16. 문제

지금은 고객 리스트가 두 개다.

```text
첫 번째 Node 고객들
두 번째 Node 고객들
```

하나로 합쳐야 한다.

------

# 17. concatenate 사용

```python
np.concatenate()
```

를 사용한다.

------

# 18. 코드

```python
frauds = np.concatenate(
    (
      mappings[(8,1)],
      mappings[(6,8)]
    ),
    axis=0
)
```

------

# 19. 왜 괄호를 두 번 쓰는가?

이 부분이 많이 헷갈린다.

```python
np.concatenate(
    (
      list1,
      list2
    ),
    axis=0
)
```

여기서:

```python
(
 list1,
 list2
)
```

는

```text
합칠 배열들의 묶음
```

이다.

------

# 20. axis=0 의미

```python
axis=0
```

은:

```text
세로 방향 결합
```

이다.

예:

```text
고객1
고객2
고객3
```

- 

```text
고객4
고객5
```

↓

```text
고객1
고객2
고객3
고객4
고객5
```

------

# 21. 왜 axis=0인가?

고객 데이터는:

```text
행(row) = 고객
```

이기 때문이다.

그래서:

```text
고객 목록 아래에 고객 목록 추가
```

해야 한다.

------

# 22. Fraud 리스트 완성

결과:

```python
frauds
```

에는

```text
잠재적 Fraud 고객 전체 목록
```

이 저장된다.

------

# 23. 아직 문제 있음

하지만 값이 여전히:

```text
0~1
```

사이 값이다.

즉:

```text
Scaling 상태
```

이다.

------

# 24. 원래 값으로 복원

```python
frauds = sc.inverse_transform(frauds)
```

------

# 25. inverse_transform 의미

처음에:

```python
X = sc.fit_transform(X)
```

를 했었다.

inverse_transform은:

```text
Scaled 값
→ 원래 값
```

으로 되돌린다.

------

# 26. 결과

이제 frauds는:

```text
원래 CustomerID
원래 Attribute 값
```

을 가진다.

예:

```text
15672829
15691234
15700112
...
```

같이 보인다.

------

# 27. 여기서 SOM의 역할 종료

우리가 해야 할 일은 끝났다.

```text
Fraud 의심 고객 리스트 생성
```

까지 완료했다.

------

# 28. 실제 은행에서는?

은행 분석가는:

```text
Fraud 고객 후보
```

를 조사한다.

특히:

```text
승인된 고객
```

을 우선적으로 본다.

왜냐하면:

```text
승인 + Fraud 가능성
```

이 가장 위험하기 때문이다.

------

# 29. 전체 흐름 정리

```text
SOM 학습
↓
MID Map 생성
↓
Outlier Node 선택
↓
win_map()
↓
Node → Customer 매핑
↓
mappings[(8,1)]
mappings[(6,8)]
↓
concatenate
↓
frauds
↓
inverse_transform
↓
실제 CustomerID 획득
```

------

# 30. 한 줄 핵심 정리

👉 `win_map()`은 **"어떤 고객이 어떤 Winning Node에 속하는지"를 저장한 dictionary를 반환하며, Outlier Node 좌표를 이용해 Fraud 후보 고객들을 추출한 뒤 `inverse_transform()`으로 실제 고객 ID를 복원하는 과정이다.**
