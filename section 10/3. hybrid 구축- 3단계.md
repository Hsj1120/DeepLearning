# 1. 이번 강의의 핵심 목표

이번 강의의 목표는:

```text
SOM 결과
→ ANN 학습 데이터로 변환
```

하는 것이다.

즉,

```text
비지도 학습
→ 지도 학습
```

으로 넘어가는 단계다.

강사가 계속 "가장 어려운 부분"이라고 말하는 이유도 바로 이것 때문이다.

------

# 2. 지금까지 상황

현재 우리는 SOM을 이용해서

```text
frauds
```

라는 리스트를 만들었다.

이 안에는:

```text
사기 의심 고객들의 정보
```

가 들어있다.

예를 들면:

```text
CustomerID 1567289
CustomerID 1569123
CustomerID 1570011
...
```

같은 고객들이다.

------

# 3. 문제점

ANN을 학습하려면:

```text
X (입력)
y (정답)
```

이 있어야 한다.

그런데 현재는:

```text
X 있음
y 없음
```

상태다.

왜냐하면 원래 데이터셋에는

```text
fraud 여부
```

가 없기 때문이다.

------

# 4. SOM이 정답을 만들어 준다

여기서 SOM 결과를 사용한다.

아이디어는 단순하다.

```text
SOM이 fraud 후보라고 판단한 고객
→ 1

나머지 고객
→ 0
```

로 두는 것이다.

즉 SOM이

```text
Pseudo Label
```

을 만들어주는 것이다.

------

# 5. 먼저 새로운 Feature Matrix 만들기

강의에서는 X 대신

```python
customers
```

라는 이름을 사용한다.

```python
customers = dataset.iloc[:, 1:].values
```

------

# 6. 왜 1: 를 사용할까?

예전에는:

```python
X = dataset.iloc[:, :-1].values
```

였다.

즉:

```text
CustomerID 포함
마지막 승인 여부 제외
```

였다.

------

# 7. 이번에는 다르다

이번에는:

```python
dataset.iloc[:, 1:]
```

를 사용한다.

즉:

```text
CustomerID 제외
나머지 전부 포함
```

이다.

------

# 8. 왜 CustomerID를 빼나?

CustomerID는 단순 번호다.

예를 들어:

```text
1567289
1569123
1570011
```

이 숫자는

```text
사기와 직접적인 관계 없음
```

이다.

ANN이 배울 정보가 아니다.

그래서 제거한다.

------

# 9. 그런데 승인 여부는 포함한다

재미있는 점은

```text
마지막 컬럼 (승인 여부)
```

를 이번에는 포함한다.

왜냐하면:

```text
승인 여부가
사기 가능성과 관련 있을 수도 있음
```

이라고 가정하기 때문이다.

------

# 10. customers 크기

이 데이터는:

```text
690명 고객
15개 feature
```

를 가진다.

즉:

```python
customers.shape
```

결과는:

```text
(690, 15)
```

이다.

------

# 11. 이제 y 만들기

새로운 정답 벡터를 만든다.

이름은:

```python
is_fraud
```

이다.

------

# 12. 처음에는 전부 0으로 시작

```python
is_fraud = np.zeros(len(dataset))
```

뜻:

```text
[0,0,0,0,0,0,0,0,...]
```

690개 생성

------

# 13. 왜 전부 0으로 만들까?

처음에는

```text
모든 고객이 정상 고객
```

이라고 가정하는 것이다.

즉:

```text
fraud 아님
```

으로 시작한다.

------

# 14. 그 다음 Fraud 고객만 1로 바꿈

이제 SOM이 찾은 Fraud 리스트를 사용한다.

```python
for i in range(len(dataset)):
```

690명의 고객을 전부 확인한다.

------

# 15. 고객 ID 가져오기

```python
dataset.iloc[i, 0]
```

의 의미:

```text
i번째 고객의
CustomerID
```

이다.

왜냐하면:

```text
0번 컬럼
=
CustomerID
```

이기 때문이다.

------

# 16. Fraud 리스트에 있는지 확인

```python
if dataset.iloc[i,0] in frauds:
```

뜻:

```text
현재 고객 ID가
fraud 리스트 안에 있나?
```

이다.

------

# 17. 있으면 1

```python
is_fraud[i] = 1
```

즉:

```text
사기 의심 고객
```

으로 표시한다.

------

# 18. 최종 결과

예를 들어:

```text
690명 중

37명 fraud
653명 normal
```

이라면

```python
is_fraud
```

는

```text
[0,0,0,1,0,0,0,1,0,...]
```

같이 된다.

------

# 19. 드디어 지도학습 데이터 완성

이제 ANN 입장에서 보면:

```text
입력 X = customers
정답 y = is_fraud
```

이다.

즉:

```python
customers
→
is_fraud
```

관계를 학습할 수 있게 된다.

------

# 20. Feature Scaling

ANN 학습 전 다시 스케일링한다.

```python
from sklearn.preprocessing import StandardScaler

sc = StandardScaler()

customers = sc.fit_transform(customers)
```

------

# 21. 왜 다시 Scaling?

SOM에서는:

```python
MinMaxScaler
```

를 썼다.

ANN에서는 보통:

```python
StandardScaler
```

를 더 많이 사용한다.

왜냐하면 신경망이 학습하기 편하기 때문이다.

------


였다.

하지만 이번 데이터는 단순하다.

# 24. 은닉층 단순화

# 25. Epoch도 줄인다

기존:


이번:

```python
epochs = 2
```

# 26. 왜 2번만?

데이터가:

```text
690개
```

밖에 없고

그래서:

```text
2 Epoch
```

만으로도 충분하다고 설명한다.

------

# 27. 학습 실행

```python
classifier.fit(
    customers,
    is_fraud,
    batch_size=1,
    epochs=2
)
```

이제 ANN이 학습한다.

------

# 28. 이번 강의의 진짜 핵심

사실 ANN 구조는 중요하지 않다.

이번 강의의 핵심은:

```text
SOM 결과를 이용해
지도학습용 y 생성
```

이다.

이게 Hybrid Model의 핵심 아이디어다.

------

# 29. 전체 흐름 정리

```text
Credit Card Data
        ↓
       SOM
        ↓
Fraud 후보 37명 발견
        ↓
Fraud 후보 = 1
나머지 = 0
        ↓
is_fraud 생성
        ↓
customers + is_fraud
        ↓
ANN 학습
```

------

# 30. 한 줄 핵심 정리

👉 이번 강의는 **SOM이 찾아낸 Fraud 후보 고객들을 1로, 나머지를 0으로 표시한 `is_fraud` 벡터를 만들어 ANN 학습용 정답 데이터(y)로 사용하는 과정**이다.
