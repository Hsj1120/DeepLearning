# 1. 이번 강의 소개.

이번 강의는 이 과정의 마지막 프로젝트 중 하나인 **Mega Case Study** 이다.

강의 제목 그대로:

```text
Hybrid Deep Learning Model
```

즉,

**두 개의 딥러닝 모델을 결합한 하이브리드 모델** 을 만드는 것이 목표이다.

------

# 2. 지금까지 무엇을 배웠을까?

지금까지 우리는 여러 딥러닝 모델을 배웠다.

그중 이번 프로젝트에서 사용할 모델은 두 개이다.

```text
Part 1 → Artificial Neural Network (ANN)
Part 4 → Self-Organizing Map (SOM)
```

이번 프로젝트는 이 두 모델을 하나로 연결한다.

------

# 3. 프로젝트 목표

최종 목표는:

```text
각 고객이 사기를 쳤을 확률을 예측하는 것
```

이다.

즉 단순히:

```text
사기다
사기가 아니다
```

를 찾는 것이 아니다.

대신:

```text
고객 A → 0.03
고객 B → 0.15
고객 C → 0.72
```

처럼

**사기 가능성(Fraud Probability)**

을 예측한다.

------

# 4. 왜 SOM만으로는 부족할까?

우리가 이전에 만든 SOM은 비지도 학습(Unsupervised Learning)이다.

즉:

```text
패턴 찾기
이상치 찾기
Outlier 찾기
```

는 잘한다.

하지만:

```text
이 고객이 사기일 확률은 몇 %인가?
```

를 계산하지는 못한다.

------

# 5. SOM이 해주는 역할

SOM은:

```text
수상한 고객 찾기
```

를 담당한다.

즉:

```text
정상 고객들과 다른 패턴
→ Outlier
→ Fraud 의심 고객
```

을 찾아준다.

------

# 6. 하지만 확률은 없다

예를 들어 SOM 결과가:

```text
고객 100명 중
5명 suspicious
```

이라고 하자.

그러면 SOM은:

```text
이 5명이 수상함
```

이라고만 말한다.

하지만:

```text
누가 가장 위험한가?
```

는 알려주지 않는다.

------

# 7. 그래서 ANN을 추가한다

여기서 ANN이 등장한다.

이번 프로젝트의 핵심 아이디어는:

```text
SOM으로 의심 고객 생성
→ ANN으로 사기 확률 예측
```

이다.

------

# 8. Hybrid Model 구조

전체 구조는 다음과 같다.

```text
Credit Card Applications Dataset
                ↓
             SOM
                ↓
      Fraud 의심 고객 탐지
                ↓
         Fraud Label 생성
                ↓
             ANN
                ↓
     Fraud Probability 예측
```

------

# 9. 첫 번째 단계

먼저 SOM을 사용한다.

이 부분은 이미 우리가 했다.

```text
데이터 로딩
→ Feature Scaling
→ SOM 학습
→ MID 계산
→ Outlier Node 찾기
→ Fraud 의심 고객 추출
```

이다.

새로운 내용은 거의 없다.

------

# 10. SOM 결과를 Label로 사용

여기서 중요한 아이디어가 나온다.

원래 데이터셋에는:

```text
Fraud 여부
```

가 존재하지 않는다.

즉:

```text
y 없음
```

이다.

그래서 지도학습을 할 수 없다.

------

# 11. SOM이 가짜 정답을 만들어준다

SOM이 찾은 Fraud 후보를:

```text
Fraud = 1
```

로 둔다.

나머지는:

```text
Fraud = 0
```

으로 둔다.

즉 SOM이:

```text
Pseudo Label
```

을 만들어주는 것이다.

------

# 12. Pseudo Label이란?

Pseudo Label은:

```text
진짜 정답은 아니지만
모델이 추정한 정답
```

이다.

예를 들어:

```text
Customer 1 → Fraud 1
Customer 2 → Fraud 0
Customer 3 → Fraud 0
Customer 4 → Fraud 1
```

처럼 만든다.

------

# 13. 이제 지도학습 가능

Pseudo Label이 생기면:

```text
X = 고객 정보
y = Fraud 여부
```

가 된다.

그러면 ANN을 학습시킬 수 있다.

------

# 14. ANN의 역할

ANN은 이제:

```text
고객 정보
→ Fraud 확률
```

을 학습한다.

즉:

```text
입력
→ 고객 특징

출력
→ Fraud Probability
```

이다.

------

# 15. 왜 Hybrid인가?

Hybrid라는 이름을 쓰는 이유는:

비지도 학습과 지도 학습을 같이 사용하기 때문이다.

```text
SOM
→ Unsupervised Learning

ANN
→ Supervised Learning
```

이다.

------

# 16. 전체 흐름

프로젝트 전체를 한 번에 보면:

```text
신용카드 신청 데이터
        ↓
      SOM
        ↓
Fraud 의심 고객 탐지
        ↓
Pseudo Label 생성
        ↓
      ANN
        ↓
Fraud 확률 계산
        ↓
위험도 순위 생성
```

이다.

------

# 17. 최종 결과

최종적으로는:

```text
Customer 2458 → 0.002
Customer 6314 → 0.015
Customer 8291 → 0.127
Customer 9142 → 0.342
```

처럼 나온다.

즉:

```text
사기 확률 순위
```

를 얻게 된다.

------

# 18. 왜 확률이 작을까?

강의에서 미리 이야기한다.

결과 확률은 매우 작게 나올 수 있다.

예를 들어:

```text
0.001
0.005
0.012
```

같이 나올 수 있다.

왜냐하면 SOM이 찾은 Fraud 수 자체가 적기 때문이다.

------

# 19. 중요한 것은 절대값이 아니다

중요한 것은:

```text
0.02냐
0.03이냐
```

가 아니다.

중요한 것은:

```text
누가 더 위험한가?
```

이다.

즉:

```text
Ranking
```

이 핵심이다.

------

# 20. 이번 프로젝트의 진짜 의미

이 프로젝트는 단순히 ANN이나 SOM을 배우는 것이 아니다.

실제 현업에서도 자주 쓰이는:

```text
비지도 모델로 이상치 탐지
→ 결과를 지도학습에 활용
```

하는 아이디어를 경험하는 것이다.

------

# 21. 한 줄 핵심 정리

👉 이번 Mega Case Study는 **SOM으로 Fraud 의심 고객을 찾고, 그 결과를 Label로 사용해 ANN을 학습시킨 뒤, 각 고객의 Fraud 확률을 예측하는 Hybrid Deep Learning 프로젝트**이다.
