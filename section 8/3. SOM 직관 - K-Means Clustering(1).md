# 1. 이번 강의의 목적

**왜 K-Means Clustering을 다시 봐야 하는가?**

를 설명하는 것이다.

------

# 2. K-Means를 보는 이유

K-Means를 다시 보는 이유는 간단하다.

**Self-Organizing Maps를 이해하는 데 도움이 되기 때문**이다.

즉:

- K-Means를 이미 배운 사람은 복습
- 처음 보는 사람은 기본 개념 학습

이 목적이다.

------

# 3. K-Means와 SOM은 같은가?

아니다.

K-Means와 SOM은 완전히 같은 알고리즘은 아니다.

K-Means를 안다고 해서
SOM이 바로 완벽하게 이해되는 것도 아니다.

👉 하지만:

K-Means는 SOM을 이해하기 위한
**준비 운동** 역할을 한다.

------

# 4. K-Means에서 중요한 느낌

![8-1](../img/8-1.png)

K-Means에서는 군집화한 결과의 중심점이 움직인다.

이 중심점을: **Centroid** 라고 한다.

Centroid는 데이터 포인트들에 의해 이리저리 움직인다.

쉽게 말하면:

- 데이터가 중심점을 끌어당김
- 중심점이 위치를 바꿈
- 다시 데이터와의 관계를 계산함
- 또 중심점이 이동함

이런 과정이 반복된다.

------

# 5. 왜 이게 SOM과 비슷할까?

SOM에서도 비슷한 느낌이 나온다.

SOM에서도 노드들이
데이터에 의해 영향을 받는다.

즉:

👉 데이터가 노드를 끌어당기고
👉 노드들이 위치나 값이 조정되고
👉 비슷한 데이터끼리 가까워지는 구조가 만들어진다.

그래서 K-Means를 보면
SOM에서 일어나는 움직임을 이해하기 쉬워진다.

------

# 6. K-Means는 비지도 학습이다

또 하나 중요한 점은:

K-Means도 **비지도 학습 알고리즘**이라는 것이다.

즉, 정답 label이 없다.

데이터만 보고
스스로 그룹을 찾는다.

다만 K-Means는 신경망은 아니다.

정리하면:

- K-Means = 비지도 머신러닝 알고리즘
- SOM = 비지도 딥러닝 / 신경망 계열 알고리즘

------

# 7. K-Means를 다시 보는 핵심 이유

K-Means를 다시 보는 이유는 두 가지이다.

1. SOM의 움직임을 이해하기 위해
2. 비지도 학습의 감각을 익히기 위해

------

# 8. 한 줄 핵심 정리

👉 K-Means는 SOM과 같지는 않지만,
**SOM에서 노드가 데이터에 의해 움직이고 조정되는 느낌을 이해하게 해주는 준비 단계**이다.
