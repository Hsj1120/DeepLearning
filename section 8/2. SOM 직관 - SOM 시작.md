# 1. Self-Organizing Maps 시작

**Self-Organizing Maps**, 즉 **SOM**에 대해 배운다.

한국어로는 **자기조직화 지도**라고 한다.

------

# 2. 이번 파트의 특징

이번 파트는 꽤 흥미로운 내용이다.

그리고 비교적 쉬운 편이다.

하지만 SOM만의 특징과 세부적인 개념이 있다.

그래서 하나씩 천천히 살펴보면 된다.

------

# 3. 지금까지 배운 딥러닝

우리는 지금까지 다음 내용을 배웠다.

- Artificial Neural Networks
- Convolutional Neural Networks
- Recurrent Neural Networks

즉:

- ANN
- CNN
- RNN

을 배웠다.

이것들은 모두 **Supervised Deep Learning**, 즉 **지도 학습 기반 딥러닝**에 해당한다.

------

# 4. 이제 배우는 것은 비지도 학습

오늘부터 배우는 SOM은 다르다.

SOM은 **Unsupervised Deep Learning**

즉 **비지도 학습 기반 딥러닝**에 해당한다.

👉 핵심 차이:

기존 ANN, CNN, RNN은 정답(label)이 있는 데이터를 사용했다.

하지만 SOM은 정답(label)이 없는 데이터를 사용한다.

------

# 5. SOM을 만든 사람

Self-Organizing Maps는 1980년대에 **Teuvo Kohonen**이라는 사람이 만들었다.

그래서 SOM은 가끔: **Kohonen Map** 이라고도 불린다.

------

# 6. SOM은 어디에 쓰일까?

SOM은 주로: **차원 축소**에 사용된다.

차원 축소란:

👉 데이터의 컬럼 수를 줄이는 것

이다.

예를 들어 데이터에 컬럼이 너무 많다고 해보자.

- 20개
- 30개
- 100개
- 그 이상

이렇게 컬럼이 많으면 사람이 직접 보기 어렵다.

이때 SOM을 사용하면 복잡한 데이터를 **2차원 지도 형태**로 줄여서 볼 수 있다.

------

# 7. SOM의 핵심 목적

SOM의 목적은 **많은 컬럼을 가진 데이터를2차원 지도 형태로 표현하는 것**

즉:

```text
많은 컬럼을 가진 데이터
→ SOM
→ 2차원 지도
```

이런 흐름이다.

👉 핵심:

복잡한 데이터를 사람이 보기 쉬운 형태로 바꿔준다.

------

# 8. 예시: 국가별 빈곤 상태 분석

강의에서는 한 논문 예시를 보여준다.

이 예시는 SOM을 천문학에 적용한 논문에서 가져온 시각화이지만, 강의에서는 국가별 빈곤 상태를 설명하는 SOM도 보여준다.

여기서는 세계 여러 나라를 빈곤 상태와 번영 상태에 따라 묶었다.

사용된 데이터는:

- 국가들 = 행(row)
- 여러 지표 = 열(column)

형태이다.

이 예시에서는 * 39개의 지표**가 사용되었다.

------

# 9. 39개의 지표란?

여기서 지표란 나라의 상태를 설명하는 값이다.

예를 들면:

- 삶의 질
- 건강 상태
- 영양 상태
- 교육 서비스
- 의료 지출
- 교육 지출
- GDP 비율
- 물가 상승률

같은 것들이다.

즉, 각 나라를 설명하는 여러 숫자 정보라고 보면 된다.

------

# 10. 왜 SOM이 필요한가?

만약 200개 이상의 국가가 있고,
각 국가마다 39개의 지표가 있다면?

데이터는 대략 이런 느낌이다.

```text
200개 이상의 행
39개의 열
```

이 데이터는 그대로 보면 한눈에 이해하기 어렵다.

👉 문제:

39차원 데이터를 사람이 직접 시각화하기 어렵다.

👉 해결:

SOM을 사용해서 2차원 지도 형태로 줄인다.

------

# 11. SOM 결과는 어떻게 보일까?

SOM 결과에서는
비슷한 나라끼리 가까이 배치된다.

예를 들어:

- 벨기에
- 스웨덴
- 일본
- 스페인
- 그리스

같은 나라들이 비슷한 영역에 배치된다.

이쪽은 상대적으로
빈곤 문제가 덜 심각한 국가들이다.

반대로 다른 쪽에는:

- 에티오피아
- 짐바브웨로 보이는 국가
- 과테말라로 보이는 국가

같은 나라들이 배치된다.

이쪽은 상대적으로
빈곤 문제가 더 심각한 국가들이다.

------

# 12. 중요한 점: SOM은 스스로 묶는다

여기서 가장 중요한 점은:

SOM은 누가 정답을 알려주지 않았다는 것이다.

즉, 알고리즘에게:

```text
벨기에와 스웨덴은 비슷해
에티오피아는 다른 그룹이야
```

라고 알려준 것이 아니다.

그냥 데이터만 줬다.

그러면 SOM이 스스로 판단한다.

👉 “이 나라들은 지표가 비슷하네.”
👉 “이 나라들은 서로 가까이 두면 되겠네.”
👉 “이 나라들은 다른 쪽에 배치해야겠네.”

이렇게 학습한다.

------

# 13. 그래서 비지도 학습이다

SOM은 정답(label)이 없다.

데이터만 보고
스스로 패턴을 찾는다.

그래서:

**Unsupervised Learning Algorithm**

즉:

**비지도 학습 알고리즘**

이라고 한다.

------

# 14. SOM 결과를 세계 지도에 표시하기

SOM에서 나온 색깔을
실제 세계 지도에 표시할 수도 있다.

그러면 더 이해하기 쉬워진다.

예를 들어:

- 빈곤 문제가 덜 심각한 국가
- 빈곤 문제가 심각한 국가
- 중간 정도의 국가

를 색깔로 구분해서 볼 수 있다.

👉 이렇게 하면
사람이 훨씬 직관적으로 이해할 수 있다.

------

# 15. SOM이 잘한 점

SOM은 데이터를 보고
국가들을 꽤 잘 구분했다.

예를 들어:

- 서유럽과 동유럽의 차이
- 선진국과 개발도상국의 차이
- 빈곤 문제가 심각한 지역과 덜 심각한 지역

이런 차이를 어느 정도 잘 잡아냈다.

👉 즉:

SOM은 정답 없이도
데이터 안의 구조를 찾아낸 것이다.

------

# 16. SOM은 어디에 더 쓸 수 있을까?

SOM은 국가 데이터에만 쓰는 것이 아니다.

다양한 데이터에 적용할 수 있다.

예를 들면:

- 질병 데이터
- 장비 종류
- 판매 상품
- 재고 데이터
- 주식 데이터
- 고객 데이터

등에도 사용할 수 있다.

👉 핵심:

컬럼이 많고 복잡한 데이터를 비슷한 것끼리 묶어서 한눈에 이해하고 싶을 때 사용한다.

------

# 17. 전체 흐름 정리

이번 예시의 흐름은 이렇다.

```text
세계은행 데이터
→ 여러 국가의 다양한 지표 수집
→ SOM에 입력
→ 비슷한 국가끼리 가까이 배치
→ 2차원 지도 생성
→ 실제 세계 지도에 색깔로 표시
```

------

# 18. SOM의 장점

SOM을 사용하면:

- 복잡한 데이터를 쉽게 볼 수 있음
- 비슷한 데이터끼리 묶을 수 있음
- 정답 없이도 패턴을 찾을 수 있음
- 고차원 데이터를 2차원으로 줄일 수 있음
- 사람이 이해하기 쉬운 시각화가 가능함
