# 📌 Part 2-3: 추가 LSTM Layer 쌓기 (Stacked LSTM 완성)

이번 단계에서는:

> 두 번째, 세 번째, 네 번째 LSTM Layer

를 추가해서:

> 진짜 “Stacked LSTM”

구조를 완성한다.

그리고 각 층마다:

```python
Dropout Regularization
```

도 함께 추가한다.

------

# 1. 이번 단계의 핵심 목표

이전 단계까지는:

```python
Input
 ↓
LSTM
 ↓
Dropout
```

구조였다.

------

이번 단계에서는 이것을:

```python
Input
 ↓
LSTM
 ↓
Dropout
 ↓
LSTM
 ↓
Dropout
 ↓
LSTM
 ↓
Dropout
 ↓
LSTM
 ↓
Dropout
```

처럼 확장한다.

즉:

```python
총 4개의 LSTM Layer
```

를 쌓는다.

------

# 2. 왜 여러 개를 쌓는가?

LSTM 하나만으로는:

```python
복잡한 시계열 패턴 학습 한계
```

가 있을 수 있다.

------

그래서 여러 층을 쌓아:

- 더 깊은 특징 추출
- 장기 패턴 학습 강화
- 복잡한 시계열 구조 표현

을 가능하게 만든다.

------

# 3. 전체 코드 구조

## 두 번째 LSTM

```python
regressor.add(
    LSTM(
        units = 50,
        return_sequences = True
    )
)

regressor.add(Dropout(0.2))
```

------

## 세 번째 LSTM

```python
regressor.add(
    LSTM(
        units = 50,
        return_sequences = True
    )
)

regressor.add(Dropout(0.2))
```

------

## 네 번째 LSTM

```python
regressor.add(
    LSTM(
        units = 50
    )
)

regressor.add(Dropout(0.2))
```

------

# 4. 두 번째 LSTM Layer

강의에서 가장 먼저 강조하는 부분:

```python
input_shape 제거
```

이다.

------

# 5. 왜 input_shape가 필요 없을까?

첫 번째 LSTM에서는:

```python
input_shape=(60,1)
```

를 직접 지정했다.

왜냐하면:

> 첫 번째 layer는 입력 데이터 구조를 알아야 하기 때문

이다.

------

# 6. 그런데 이후 Layer는?

두 번째 LSTM부터는:

```python
이전 layer 출력 자동 인식
```

한다.

즉:

```python
첫 번째 LSTM 출력
 ↓
두 번째 LSTM 입력
```

이 자동 연결된다.

------

# 7. 그래서 input_shape 제거

즉:

```python
input_shape 생략 가능
```

------

# 8. 내부적으로 어떻게 연결될까?

첫 번째 LSTM:

```python
(60,1)
```

입력

↓

출력:

```python
(60,50)
```

------

# 9. 왜 (60,50)이 되는가?

우리가:

```python
units = 50
```

를 사용했기 때문이다.

------

즉 timestep마다:

```python
50차원 hidden state
```

를 출력한다.

------

# 10. 따라서 두 번째 LSTM 입력은 자동 결정

즉 두 번째 LSTM은:

```python
(60,50)
```

을 자동 입력으로 받는다.

그래서:

```python
input_shape 안 써도 됨
```

------

# 11. 두 번째 LSTM도 return_sequences=True

## 코드

```python
return_sequences = True
```

------

# 12. 이유 복습

우리는:

```python
세 번째 LSTM
```

을 또 추가할 예정이다.

따라서:

```python
sequence 전체 전달 필요
```

------

즉:

```python
모든 timestep 출력 유지
```

해야 한다.

------

# 13. 세 번째 LSTM도 동일

세 번째 LSTM은:

```python
두 번째와 완전히 동일
```

하다.

------

## 코드

```python
regressor.add(
    LSTM(
        units = 50,
        return_sequences = True
    )
)
```

------

# 14. 왜 계속 50 neurons 사용?

강의에서 계속 강조하는 부분:

```python
high dimensionality
```

이다.

------

즉:

```python
복잡한 주가 패턴 표현
```

을 위해:

```python
충분한 뉴런 수 유지
```

하는 것이다.

------

# 15. 모델 차원이 높아진다는 의미

층이 깊어지고 뉴런 수가 많아질수록:

```python
표현 가능한 패턴 수 증가
```

한다.

------

즉:

```python
상승 패턴
하락 패턴
장기 추세
단기 변동
```

등을 더 잘 학습 가능

------

# 16. Dropout도 계속 추가

각 LSTM 뒤에는:

```python
Dropout(0.2)
```

를 추가한다.

------

# 17. 왜 매 layer마다 Dropout?

층이 깊어질수록:

```python
Overfitting 위험 증가
```

하기 때문이다.

------

따라서:

```python
매 layer마다 regularization
```

을 수행한다.

------

# 18. 이제 마지막 네 번째 LSTM

여기서 중요한 변화가 하나 생긴다.

------

# 19. return_sequences 제거

## 코드

```python
LSTM(units = 50)
```

------

# 20. 왜 True 안 쓰는가?

네 번째 LSTM 뒤에는:

```python
더 이상 LSTM 없음
```

------

즉:

```python
Output Layer(Dense)
```

만 남아 있다.

------

# 21. 따라서 sequence 전체 필요 없음

이제는:

```python
마지막 hidden state 하나만 필요
```

하다.

------

# 22. return_sequences=False 상태

사실:

```python
return_sequences=False
```

가 기본값(default)이다.

------

따라서:

```python
굳이 안 써도 됨
```

------

# 23. 출력 구조 차이

## True일 때

```python
[
 h1,
 h2,
 h3,
 ...
 h60
]
```

------

## False일 때

```python
h60 하나만 출력
```

------

# 24. 마지막 LSTM의 역할

마지막 LSTM은:

```python
전체 시계열을 요약한 최종 기억(hidden state)
```

를 만든다.

------

이 값이 이후:

```python
Dense Output Layer
```

로 전달된다.

------

# 25. 현재까지 완성된 구조

현재 구조:

```python
Input
 ↓
LSTM(50)
 ↓
Dropout
 ↓
LSTM(50)
 ↓
Dropout
 ↓
LSTM(50)
 ↓
Dropout
 ↓
LSTM(50)
 ↓
Dropout
```

------

# 26. 아직 남은 것

이제 마지막으로 필요한 것은:

```python
출력층(Dense)
```

뿐이다.

------

# 27. 왜 Dense가 필요한가?

우리는 최종적으로:

```python
주가 1개 값
```

을 예측해야 한다.

따라서 마지막 hidden state를:

```python
실제 숫자 값
```

으로 변환할 출력층이 필요하다.

------
