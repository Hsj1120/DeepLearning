# 📌 RNN 예측값 만들기 - X_test 생성과 예측 수행

## 1. 이번 단계의 목표

이전 단계에서는 2017년 1월 주가 예측에 필요한 `inputs`를 만들었다.

이번 단계에서는 그 `inputs`를 이용해서:

```python
X_test
```

를 만들고, 최종적으로:

```python
predicted_stock_price
```

를 얻는다.

즉, 이번 단계는:

> RNN 모델에 넣을 3차원 입력 데이터를 만들고 실제 예측을 수행하는 과정이다.

------

# 2. 왜 X_test를 만들어야 할까?

RNN/LSTM은 입력 데이터를 그냥 1차원이나 2차원으로 받지 않는다.

이전에 학습할 때도 입력 구조는 다음과 같았다.

```python
(samples, timesteps, features)
```

즉:

```python
몇 개의 데이터인지,
각 데이터가 몇 일치를 보는지,
각 날짜마다 feature가 몇 개인지
```

를 가진 3차원 구조가 필요하다.

------

# 3. X_test 만들기

## ✔ 코드

```python
X_test = []

for i in range(60, 80):
    X_test.append(inputs[i-60:i, 0])

X_test = np.array(X_test)
```

------

# 4. range(60, 80)의 의미

여기서 `inputs`에는 총 80개 데이터가 들어 있다.

```python
60일치 과거 데이터 + 2017년 1월 test 데이터 20개
```

그래서 반복문은:

```python
range(60, 80)
```

을 사용한다.

의미는:

```python
60부터 79까지 반복
```

즉 총 20번 반복한다.

왜 20번이냐면:

> 2017년 1월 거래일이 20개이기 때문이다.

------

# 5. X_test에 들어가는 구조

```python
X_test.append(inputs[i-60:i, 0])
```

이 코드는 매번 이전 60일 데이터를 가져온다.

예를 들어:

```python
i = 60
```

이면:

```python
inputs[0:60]
```

을 가져온다.

이건 2017년 1월 3일을 예측하기 위한 이전 60일 데이터이다.

```python
i = 61
```

이면:

```python
inputs[1:61]
```

을 가져온다.

이건 2017년 1월 4일을 예측하기 위한 이전 60일 데이터이다.

즉:

```python
60일치 데이터 → 하루 예측
```

구조를 20개 만드는 것이다.

------

# 6. X_test의 최종 형태

처음에는 리스트로 만들지만:

```python
X_test = np.array(X_test)
```

를 통해 NumPy 배열로 바꾼다.

이때 형태는:

```python
(20, 60)
```

이다.

의미는:

| 값   | 의미                                    |
| ---- | --------------------------------------- |
| 20   | 예측할 날짜 수                          |
| 60   | 각 날짜를 예측할 때 보는 과거 주가 개수 |

즉:

```python
20개의 sample
각 sample은 60일치 주가
```

이다.

------

# 7. 하지만 아직 RNN 입력 형태가 아니다

RNN/LSTM은 다음 형태를 요구한다.

```python
(samples, timesteps, features)
```

현재 `X_test`는:

```python
(20, 60)
```

이다.

여기에는 feature 차원이 빠져 있다.

그래서 다시 reshape 해야 한다.

------

# 8. 3차원 형태로 reshape

## ✔ 코드

```python
X_test = np.reshape(X_test, (X_test.shape[0], X_test.shape[1], 1))
```

## ✔ 의미

이 코드는 `X_test`를 다음 형태로 바꾼다.

```python
(20, 60, 1)
```

의미는:

| 차원 | 의미      |
| ---- | --------- |
| 20   | samples   |
| 60   | timesteps |
| 1    | features  |

즉:

```python
20개의 예측 데이터
각 데이터는 과거 60일을 보고
각 날짜마다 Open 주가 1개 feature를 가진다
```

------

# 9. 예측 수행하기

## ✔ 코드

```python
predicted_stock_price = regressor.predict(X_test)
```

## ✔ 의미

학습된 RNN 모델인 `regressor`에 `X_test`를 넣어서 2017년 1월 주가를 예측한다.

결과는:

```python
predicted_stock_price
```

에 저장된다.

이 값은 총 20개이다.

왜냐하면 2017년 1월 거래일이 20개이기 때문이다.

------

# 10. 그런데 예측값은 아직 scaled 값이다

중요한 점이 있다.

모델은 학습할 때 scaled 된 값을 기준으로 학습했다.

즉, 모델은 원래 주가인:

```python
780
790
800
```

같은 값을 예측하는 것이 아니라,

```python
0.72
0.75
0.78
```

같은 scaled 값을 예측한다.

그래서 예측 결과도 아직 0~1 사이 값이다.

------

# 11. 원래 주가 단위로 되돌리기

## ✔ 코드

```python
predicted_stock_price = sc.inverse_transform(predicted_stock_price)
```

## ✔ 의미

`inverse_transform()`은 scaling된 값을 다시 원래 주가 단위로 되돌리는 함수이다.

예를 들어:

```python
0.75
```

같은 값을 다시:

```python
790
```

같은 실제 주가 값으로 되돌린다.

------

# 12. 왜 inverse_transform이 필요할까?

나중에 실제 주가와 예측 주가를 비교해야 한다.

실제 주가는 원래 단위이다.

```python
real_stock_price = 780, 790, 800 ...
```

그런데 예측값이 scaled 값이면:

```python
predicted_stock_price = 0.72, 0.75, 0.78 ...
```

단위가 서로 다르다.

그래서 비교가 불가능하다.

따라서 예측값을 다시 원래 주가 단위로 되돌려야 한다.

------

# 13. 전체 코드 최종 정리

```python
X_test = []

for i in range(60, 80):
    X_test.append(inputs[i-60:i, 0])

X_test = np.array(X_test)

X_test = np.reshape(X_test, (X_test.shape[0], X_test.shape[1], 1))

predicted_stock_price = regressor.predict(X_test)

predicted_stock_price = sc.inverse_transform(predicted_stock_price)
```

------

# 🎯 최종 핵심 정리

- `inputs`에서 60일씩 잘라서 `X_test`를 만든다.
- 2017년 1월 거래일은 20개이므로 `range(60, 80)`을 사용한다.
- 처음 `X_test`의 형태는 `(20, 60)`이다.
- RNN/LSTM은 3차원 입력을 요구하므로 `(20, 60, 1)`로 reshape 한다.
- `regressor.predict(X_test)`로 예측값을 만든다.
- 예측값은 scaled 값이므로 `inverse_transform()`으로 원래 주가 단위로 되돌린다.
