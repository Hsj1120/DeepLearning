# 📌 RNN 예측 결과 만들기 - 실제 주가 데이터 가져오기

## 1. 이번 단계의 목표

이번 파트에서는 RNN 모델이 예측한 값과 실제 Google 주가를 비교한다.

전체 과정은 크게 3단계이다.

1. **2017년 실제 Google 주가 가져오기**
2. **2017년 Google 주가 예측값 만들기**
3. **실제값과 예측값 시각화하기**

이번 강의에서는 그중 첫 번째 단계인:

> 2017년 1월 실제 Google 주가 데이터를 가져오는 과정

을 수행한다.

------

# 2. 사용할 데이터

이번에는 학습 데이터가 아니라 테스트 데이터를 사용한다.

```python
Google_Stock_Price_Test.csv
```

이 파일에는 2017년 1월 Google 주가 데이터가 들어 있다.

즉, 모델이 예측해야 할 실제 정답 데이터라고 볼 수 있다.

------

# 3. 테스트 데이터 불러오기

## ✔ 코드

```python
dataset_test = pd.read_csv('Google_Stock_Price_Test.csv')
```

## ✔ 의미

`pandas`의 `read_csv()` 함수를 사용해서 CSV 파일을 불러온다.

결과는 DataFrame 형태로 저장된다.

```python
dataset_test
```

는 2017년 1월 Google 주가 데이터가 들어 있는 DataFrame이다.

------

# 4. 실제 주가 컬럼 선택하기

## ✔ 코드

```python
real_stock_price = dataset_test.iloc[:, 1:2].values
```

## ✔ 의미

테스트 데이터에서 Open 주가 컬럼만 선택한다.

```python
.iloc[:, 1:2]
```

는 전체 행에서 1번 컬럼만 선택하되, 2차원 형태를 유지한다는 뜻이다.

그리고 마지막에:

```python
.values
```

를 붙여서 NumPy 배열로 변환한다.

------

# 5. 변수 이름을 real_stock_price로 쓰는 이유

기존에는 학습 데이터에서:

```python
training_set
```

이라는 이름을 사용했다.

하지만 이번 데이터는 학습용이 아니라 실제 정답값이다.

그래서 더 명확하게:

```python
real_stock_price
```

라고 이름을 붙인다.

즉:

> 실제 Google 주가 데이터

라는 의미이다.

------

# 6. 나중에 만들 예측값

다음 단계에서는 모델이 예측한 값을 만들고:

```python
predicted_stock_price
```

라는 변수에 저장한다.

따라서 나중에는 두 값을 비교하게 된다.

```python
real_stock_price
predicted_stock_price
```

즉:

| 변수                    | 의미                     |
| ----------------------- | ------------------------ |
| `real_stock_price`      | 실제 Google 주가         |
| `predicted_stock_price` | RNN이 예측한 Google 주가 |

------

# 7. 데이터 개수

2017년 1월 데이터는 총 20개이다.

이유는 주식 시장이 매일 열리는 것이 아니라:

- 토요일 제외
- 일요일 제외
- 실제 거래일만 포함

되기 때문이다.

그래서 1월 전체 날짜가 아니라, 금융 거래일 기준으로 20개의 주가 데이터가 존재한다.

------

# 8. 우리가 기대하는 예측 결과

실제 Google 주가를 보면:

- 초반에는 상승 추세
- 중간에 큰 spike 발생
- 이후 약간 하락하는 흐름

이 나타난다.

하지만 모델이 spike까지 완벽히 맞추는 것을 기대하면 안 된다.

왜냐하면 spike는 갑작스러운 불규칙 변화이기 때문이다.

RNN 모델이 목표로 하는 것은:

> 불규칙한 순간 변동을 정확히 맞추는 것이 아니라 전체적인 추세를 따라가는 것

이다.

------
