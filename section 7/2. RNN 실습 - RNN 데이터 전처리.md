# 📌 RNN 데이터 전처리 (Training Set Import) 정리

## 1. 이번 단계의 목표

이번 단계에서는 RNN 학습을 위해:

- 학습 데이터(Training Set)를 불러오고
- 필요한 컬럼만 선택한 뒤
- NumPy 배열 형태로 변환하는 작업을 수행한다.

즉,

> RNN이 학습 가능한 형태로 데이터를 준비하는 과정이다.

------

# 2. 왜 Training Set만 먼저 불러오는가?

이번 실습에서는:

- Train Data (2012~2016)
- Test Data (2017년 1월)

를 분리해서 사용한다.

------

## ✔ 핵심 이유

RNN은 학습 과정에서:

- Test 데이터를 절대 보면 안 된다.

즉,

> 테스트 데이터는 미래 데이터 역할을 한다.

------

## ✔ 실제 의미

모델은:

- 2012~2016 데이터만 보고 학습
- 이후 처음 보는 2017 데이터를 예측

하게 된다.

즉 실제 미래 예측 상황과 동일하게 구성하는 것이다.

------

# 4. Training Set 불러오기

## ✔ 코드

```python
dataset_train = pd.read_csv('Google_Stock_Price_Train.csv')
```

------

## ✔ 의미

CSV 파일을 읽어서:

> Pandas DataFrame 형태로 저장

한다.

------

## ✔ 결과

현재 상태:

```python
dataset_train
```

은 DataFrame이다.

즉:

- 표 형태 데이터
- 행(Row) + 열(Column) 구조

를 가진다.

------

# 5. 필요한 컬럼만 선택하기

우리가 필요한 것은:

> Open 주가(Open Price)

이다.

------

## ✔ 이유

이번 실습 목표가:

> “시작 시점의 Google 주가 예측”

이기 때문이다.

------

# 6. iloc 사용

## ✔ 코드

```python
training_set = dataset_train.iloc[:, 1:2].values
```

------

# 7. iloc 구조 이해

## ✔ 형태

```python
.iloc[행, 열]
```

------

## ✔ 의미

```python
:
```

는:

> 전체 선택

의 의미다.

------

## ✔ 따라서

```python
.iloc[:, 1:2]
```

의 의미는:

| 부분 | 의미          |
| ---- | ------------- |
| :    | 모든 행       |
| 1:2  | 1번 컬럼 선택 |

이다.

------

# 8. 왜 1만 쓰지 않고 1:2를 사용하는가?

이 부분이 매우 중요하다.

------

## ✔ 만약 이렇게 하면

```python
.iloc[:, 1]
```

결과:

```python
Vector 형태
```

즉:

1차원 배열이 된다.

------

## ✔ 하지만 이렇게 하면

```python
.iloc[:, 1:2]
```

결과:

```python
2차원 배열
```

이 된다.

------

# 9. 왜 2차원 배열이 중요한가?

딥러닝 입력 데이터는 기본적으로:

```python
(samples, features)
```

형태를 요구한다.

즉:

| 샘플 수 | 특징 수 |
| ------- | ------- |
| 1258    | 1       |

구조가 되어야 한다.

------

## ✔ 따라서

```python
1:2
```

를 사용해서:

> “열 하나짜리 DataFrame”

형태를 유지한다.

------

# 10. `.values` 의 역할

## ✔ 코드

```python
.values
```

------

## ✔ 역할

Pandas DataFrame → NumPy Array 변환

------

## ✔ 이유

Keras 기반 신경망은:

> NumPy 배열만 입력 가능

하기 때문이다.

------

# 11. 최종 결과

## ✔ training_set 구조

```python
(1258, 1)
```

------

## ✔ 의미

| 값   | 의미             |
| ---- | ---------------- |
| 1258 | 주가 데이터 개수 |
| 1    | Open Price 컬럼  |

------

# 12. 전체 흐름 정리

## ✔ Step 1

CSV 파일 불러오기

```python
pd.read_csv()
```

------

## ✔ Step 2

필요 컬럼 선택

```python
.iloc[:, 1:2]
```

------

## ✔ Step 3

NumPy 배열 변환

```python
.values
```

------

# 13. 이번 단계의 핵심 목적

이번 단계의 진짜 목적은:

> RNN이 학습 가능한 입력 형태 만들기

이다.

즉:

- DataFrame → NumPy
- 1차원 → 2차원
- 필요한 컬럼만 선택

과정을 수행한 것이다.

------

# 🎯 최종 핵심 정리

- RNN은 NumPy 배열 형태의 입력을 요구한다.
- Pandas로 CSV를 불러온 후 필요한 컬럼만 선택한다.
- `.iloc[:, 1:2]` 는 Open Price 컬럼을 2차원 형태로 유지한다.
- `.values` 를 사용해 NumPy 배열로 변환한다.
- 최종적으로 `(1258, 1)` 형태의 학습 데이터가 생성된다.
