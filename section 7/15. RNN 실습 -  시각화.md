이번 파일은 **실제 주가와 예측 주가를 그래프로 비교하고, 결과를 해석하는 단계**입니다.

------

# 📌 RNN 예측 결과 시각화하기

## 1. 이번 단계의 목표

이번 단계에서는 앞에서 만든 두 데이터를 그래프로 비교한다.

```python
real_stock_price
predicted_stock_price
```

즉:

| 변수                    | 의미                        |
| ----------------------- | --------------------------- |
| `real_stock_price`      | 2017년 1월 실제 Google 주가 |
| `predicted_stock_price` | RNN이 예측한 Google 주가    |

이 두 값을 같은 그래프에 그려서 모델이 실제 흐름을 얼마나 잘 따라갔는지 확인한다.

------

# 2. 실제 주가 그래프 그리기

## ✔ 코드

```python
plt.plot(real_stock_price, color = 'red', label = 'Real Google Stock Price')
```

## ✔ 의미

`plt.plot()`은 그래프를 그리는 함수이다.

여기서는 실제 Google 주가를 빨간색 선으로 그린다.

```python
color = 'red'
```

는 선 색깔을 빨간색으로 설정한다는 뜻이다.

```python
label = 'Real Google Stock Price'
```

는 범례에 표시될 이름이다.

------

# 3. 예측 주가 그래프 그리기

## ✔ 코드

```python
plt.plot(predicted_stock_price, color = 'blue', label = 'Predicted Google Stock Price')
```

## ✔ 의미

이번에는 RNN이 예측한 주가를 파란색 선으로 그린다.

즉:

```python
빨간색 선 → 실제 주가
파란색 선 → 예측 주가
```

로 비교한다.

------

# 4. 그래프 제목 추가하기

## ✔ 코드

```python
plt.title('Google Stock Price Prediction')
```

## ✔ 의미

그래프의 제목을 설정한다.

그래프 상단에:

```python
Google Stock Price Prediction
```

이라는 제목이 표시된다.

------

# 5. x축 이름 추가하기

## ✔ 코드

```python
plt.xlabel('Time')
```

## ✔ 의미

x축은 시간 흐름을 의미한다.

여기서는 2017년 1월 3일부터 1월 31일까지의 거래일 흐름이다.

------

# 6. y축 이름 추가하기

## ✔ 코드

```python
plt.ylabel('Google Stock Price')
```

## ✔ 의미

y축은 Google 주가 값을 의미한다.

즉, 실제 주가와 예측 주가가 어느 가격대에 있는지 보여준다.

------

# 7. 범례 표시하기

## ✔ 코드

```python
plt.legend()
```

## ✔ 의미

범례를 그래프에 표시한다.

범례는 각 선이 무엇을 의미하는지 알려준다.

예를 들어:

```python
Real Google Stock Price
Predicted Google Stock Price
```

가 그래프 안에 표시된다.

------

# 8. 그래프 출력하기

## ✔ 코드

```python
plt.show()
```

## ✔ 의미

지금까지 설정한 그래프를 화면에 출력한다.

------

# 9. 전체 코드

```python
plt.plot(real_stock_price, color = 'red', label = 'Real Google Stock Price')
plt.plot(predicted_stock_price, color = 'blue', label = 'Predicted Google Stock Price')
plt.title('Google Stock Price Prediction')
plt.xlabel('Time')
plt.ylabel('Google Stock Price')
plt.legend()
plt.show()
```

# 10. 결과 해석

그래프에서:

```python
빨간색 선 = 실제 주가
파란색 선 = 예측 주가
```

이다.

모델이 잘 예측했다면 파란색 선이 빨간색 선의 흐름을 어느 정도 따라가야 한다.

하지만 완전히 똑같이 맞추는 것은 어렵다.

특히 주가에서 갑자기 튀는 spike는 예측하기 어렵다.

------

# 11. 왜 spike는 잘 못 맞출까?

주가가 갑자기 크게 오르거나 떨어지는 구간은 보통 불규칙한 변화이다.

예를 들어:

```python
갑작스러운 뉴스
기업 실적 발표
시장 분위기 변화
외부 경제 이슈
```

같은 요인으로 인해 주가가 순간적으로 튈 수 있다.

이런 변화는 과거 60일 주가 패턴만 보고 예측하기 어렵다.

그래서 모델은 spike를 바로 따라가지 못하고 약간 늦게 반응할 수 있다.

------

# 12. lagging behind 의미

강의에서 말한 `lagging behind`는:

> 예측값이 실제값보다 늦게 따라간다

는 뜻이다.

예를 들어 실제 주가가 갑자기 확 올라갔는데, 예측값은 바로 올라가지 못하고 조금 뒤에 따라 올라가는 현상이다.

이건 RNN이 과거 데이터 기반으로 예측하기 때문에 자연스럽게 생길 수 있다.

------

# 13. 모델이 잘하는 부분

RNN은 갑작스러운 spike보다:

```python
완만한 상승 추세
완만한 하락 추세
안정적인 흐름
```

을 따라가는 데 더 강하다.

그래서 그래프에서 전체적인 상승, 하락 방향을 비슷하게 따라간다면 어느 정도 의미 있는 결과라고 볼 수 있다.

------

# 14. 결과를 어떻게 봐야 할까?

이 모델의 목표는:

```python
정확한 하루 주가를 완벽히 맞히는 것
```

이라기보다는:

```python
전체적인 추세를 따라가는 것
```

에 가깝다.

따라서 결과를 볼 때는:

- spike를 정확히 맞췄는가?
- 하루하루 가격을 완벽히 맞췄는가?

보다:

- 상승 추세를 따라가는가?
- 하락 추세를 따라가는가?
- 안정적인 구간에서 비슷한 흐름을 보이는가?

를 보는 것이 더 중요하다.
