# 📌 RNN 예측값 만들기 - 60일 입력 데이터 준비

## 1. 이번 단계의 목표

이번 단계에서는 RNN 모델을 이용해:

```python
2017년 1월 Google 주가
```

를 예측하기 위한 입력 데이터를 준비한다.

즉, 아직 실제로 예측하는 단계는 아니다.

이번 단계의 핵심은:

```python
regressor.predict()
```

에 넣을 데이터를 만드는 것이다.

# 2. 먼저 기억해야 할 것

우리가 만든 RNN은 이렇게 학습했다.

```python
과거 60일 주가 → 다음 날 주가 예측
```

즉, 모델은 하루 주가를 예측할 때마다:

```python
이전 60거래일 주가
```

를 입력으로 받는다.

예를 들어:

```python
60일치 주가 → 61번째 날 주가
```

를 맞히도록 학습한 것이다.

------

# 3. 2017년 1월 주가를 예측하려면?

2017년 1월 첫 거래일은 1월 3일이다.

그럼 2017년 1월 3일 주가를 예측하려면:

```python
2017년 1월 3일 이전 60거래일 주가
```

가 필요하다.

그런데 이 60일 데이터는 대부분:

```python
2016년 10월 ~ 2016년 12월
```

에 해당한다.

즉, test set 안에 있는 것이 아니라:

```python
training set 안에 있는 데이터
```

이다.

그래서 2017년 1월을 예측하려면 test set만 사용하면 안 되고:

```python
training set + test set
```

을 함께 사용해야 한다.

------

# 4. 왜 training set과 test set을 합칠까?

2017년 1월 전체를 예측하려면 각 날짜마다 이전 60일 데이터가 필요하다.

예를 들면:

```python
1월 3일 예측 → 2016년 이전 60거래일 필요
1월 4일 예측 → 1월 3일을 포함한 이전 60거래일 필요
1월 5일 예측 → 1월 3일, 1월 4일을 포함한 이전 60거래일 필요
```

즉, 1월 뒤쪽 날짜를 예측할수록 test set의 앞부분 실제 주가도 입력으로 필요하다.

그래서 전체 주가 흐름을 이어 붙인 데이터가 필요하다.

------

dataset_total = pd.concat((dataset_train['Open'], dataset_test['Open']), axis = 0)

```python
dataset_train['Open']

```

처럼 Open 컬럼만 선택해서 합친다.

# 7. axis = 0 의미

```python
axis = 0
```

은 세로 방향으로 합친다는 뜻이다.

즉, 행을 아래로 이어 붙인다.

| 코드       | 의미           |
| ---------- | -------------- |
| `axis = 0` | 세로 방향 결합 |
| `axis = 1` | 가로 방향 결합 |

이번에는 시간 순서대로 주가 데이터를 아래에 붙여야 하므로:

```python
axis = 0
```

을 사용한다.

# 8. inputs 만들기

## ✔ 코드

```python
inputs = dataset_total[len(dataset_total) - len(dataset_test) - 60:].values
```

이 코드가 가장 헷갈릴 수 있다.

------

# 9. len(dataset_total) - len(dataset_test)의 의미

예를 들어 데이터 개수가 다음과 같다고 하자.

```python
len(dataset_train) = 1258
len(dataset_test) = 20
len(dataset_total) = 1278
```

그러면:

```python
len(dataset_total) - len(dataset_test)
```

은:

```python
1278 - 20 = 1258
```

이다.

이 위치는 바로:

```python
test set이 시작되는 위치
```

이다.

즉, 2017년 1월 3일의 위치라고 볼 수 있다.

------

# 10. 왜 다시 -60을 할까?

모델은 2017년 1월 3일을 예측하려면 그 이전 60일이 필요하다.

그래서 test set 시작 위치에서 60일 전으로 이동해야 한다.

```python
len(dataset_total) - len(dataset_test) - 60
```

즉:

```python
test 시작 위치 - 60일
```

부터 데이터를 가져오는 것이다.

------

# 11. inputs에는 무엇이 들어갈까?

```python
inputs = dataset_total[1198:].values
```

같은 느낌이다.

즉, test 시작 60일 전부터 마지막까지 가져온다.

결과적으로 inputs에는:

```python
60일치 과거 데이터 + 2017년 1월 test 데이터 20개
```

가 들어간다.

그래서 총:

```python
80개
```

의 주가 데이터가 들어간다.

------

# 12. 왜 test 데이터도 inputs에 포함될까?

이 부분이 중요하다.

2017년 1월 3일을 예측할 때는 test 데이터가 필요 없다.

하지만 1월 4일을 예측할 때는:

```python
1월 3일 실제 주가
```

가 이전 60일 안에 포함된다.

1월 5일을 예측할 때는:

```python
1월 3일 실제 주가
1월 4일 실제 주가
```

가 이전 60일 안에 포함된다.

즉, 뒤쪽 날짜를 예측할수록 test set의 앞부분 실제값도 입력으로 사용된다.

이것은 미래 값을 미리 알려주는 것이 아니라:

> 해당 날짜 이전의 실제 과거값을 사용하는 것

이다.

------

# 13. 실제 정답값은 건드리면 안 된다

여기서 중요한 주의점이 있다.

실제 정답값인:

```python
real_stock_price
```

를 직접 scaling하면 안 된다.

왜냐하면 이 값은 나중에 예측값과 비교해야 하는 실제 주가이기 때문이다.

즉:

```python
real_stock_price
```

는 원본 그대로 유지해야 한다.

# 14. reshape 하는 이유

## ✔ 코드

```python
inputs = inputs.reshape(-1, 1)
```

처음 `inputs`를 만들면 형태가 보통 이렇게 된다.

```python
(80,)
```

이건 1차원 배열이다.

하지만 `MinMaxScaler`는 보통 2차원 배열을 기대한다.

그래서 다음처럼 바꿔준다.

```python
(80, 1)
```

즉:

```python
80개의 데이터
1개의 feature
```

형태로 바꾸는 것이다.

------

# 15. reshape(-1, 1)의 의미

```python
reshape(-1, 1)
```

에서:

```python
-1
```

은 행 개수를 자동으로 맞추라는 뜻이고,

```python
1
```

은 컬럼을 1개로 만들라는 뜻이다.

즉:

```python
inputs.reshape(-1, 1)
```

은:

```python
데이터를 한 줄짜리 배열이 아니라
세로로 긴 2차원 배열로 바꿔라
```

라는 의미이다.

------

# 16. inputs scaling 하기

## ✔ 코드

```python
inputs = sc.transform(inputs)
```

여기서는 반드시:

```python
transform()
```

만 사용해야 한다.

절대:

```python
fit_transform()
```

을 다시 쓰면 안 된다.

------

# 17. 왜 fit_transform을 쓰면 안 될까?

학습할 때 이미 scaler를 만들었다.

```python
sc = MinMaxScaler(feature_range = (0, 1))
training_set_scaled = sc.fit_transform(training_set)
```

여기서 `fit_transform()`은:

```python
training set의 최소값과 최대값을 찾고
그 기준으로 0~1 사이 값으로 변환
```

하는 작업이다.

즉, `sc`는 이미 training set 기준을 기억하고 있다.

예측할 때도 같은 기준을 써야 하므로:

```python
sc.transform(inputs)
```

을 사용한다.

------

# 18. 예측할 때도 scaling이 필요한 이유

RNN은 학습할 때 0~1 사이 값으로 학습했다.

예를 들어 학습 때 입력은 이런 형태였다.

```python
0.12
0.15
0.18
```

그런데 예측할 때 원본 주가를 그대로 넣으면:

```python
780
790
800
```

같은 값이 들어간다.

그러면 모델 입장에서는 학습 때와 전혀 다른 형태의 데이터가 들어오는 것이다.

그래서 예측 입력도 반드시 같은 방식으로 scaling해야 한다.

------

# 19. fit_transform과 transform 차이

| 함수              | 의미                  | 사용 시점    |
| ----------------- | --------------------- | ------------ |
| `fit_transform()` | 기준을 새로 찾고 변환 | training set |
| `transform()`     | 기존 기준으로 변환    | test input   |

즉:

```python
training set → fit_transform()
test input → transform()
```

이다.

# 🎯 최종 핵심 정리

- RNN은 과거 60일 주가를 보고 다음 날 주가를 예측하도록 학습했다.
-  2017년 1월 주가를 예측하려면 각 날짜 이전 60일 데이터가 필요하다.
- 2017년 1월 초의 이전 60일은 training set에 들어 있다.
- 그래서 `dataset_train`과 `dataset_test`를 합친다.
- 단, 실제 정답값인 `real_stock_price`는 원본 그대로 둔다.
- 예측에 사용할 `inputs`만 scaling한다.
- `fit_transform()`이 아니라 `transform()`을 사용한다.
- 이유는 training set에서 학습한 scaling 기준을 그대로 사용해야 하기 때문이다.
