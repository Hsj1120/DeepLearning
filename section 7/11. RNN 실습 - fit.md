# 📌 Part 2-6: RNN 학습시키기 — `fit()`

이번 단계는 Part 2의 마지막 단계이다.

지금까지는:

```python
RNN 구조 만들기
```

를 했다.

이제는 실제로:

```python
학습 데이터에 연결해서 훈련
```

을 시작한다.

------

# 1. 이번 단계 코드

```python
regressor.fit(
    X_train,
    y_train,
    epochs = 100,
    batch_size = 32
)
```

------

# 2. fit이란?

`fit()`은:

> 모델을 실제 데이터로 학습시키는 함수

이다.

즉, 지금까지 만든 RNN이:

```python
X_train을 보고
y_train을 맞추도록
```

훈련된다.

------

# 3. X_train은 무엇인가?

```python
X_train
```

은 입력 데이터이다.

형태는:

```python
(1198, 60, 1)
```

이다.

의미는:

| 값   | 의미                    |
| ---- | ----------------------- |
| 1198 | 학습 샘플 개수          |
| 60   | 과거 60일 데이터        |
| 1    | feature 1개, Open Price |

즉:

```python
과거 60일 주가를 입력
```

으로 넣는 것이다.

------

# 4. y_train은 무엇인가?

```python
y_train
```

은 정답 데이터이다.

즉:

```python
과거 60일 이후의 다음 날 주가
```

이다.

------

# 5. 학습 구조

예를 들어:

```python
X_train[0] = 1일차 ~ 60일차 주가
y_train[0] = 61일차 주가
```

이다.

다음 샘플은:

```python
X_train[1] = 2일차 ~ 61일차 주가
y_train[1] = 62일차 주가
```

이런 식이다.

------

# 6. epochs = 100

```python
epochs = 100
```

은:

> 전체 학습 데이터를 100번 반복해서 학습

한다는 뜻이다.

------

# 7. epoch 하나의 의미

1 epoch는:

```python
1198개 학습 샘플 전체를 한 번 다 학습
```

하는 것이다.

------

즉:

```python
epochs = 100
```

이면:

```python
1198개 전체 데이터 학습
× 100번 반복
```

이다.

------

# 8. 왜 100번인가?

강의에서는 실험적으로:

- 25 epochs → loss가 충분히 수렴하지 않음
- 50 epochs → 아직 부족
- 100 epochs → loss가 어느 정도 수렴

했다고 설명한다.

그래서 이 예제에서는:

```python
epochs = 100
```

을 사용한다.

------

# 9. batch_size = 32

```python
batch_size = 32
```

은:

> 1198개 데이터를 32개씩 나눠서 학습

한다는 뜻이다.

------

# 10. 실제 학습 흐름

```python
1번째 batch: 32개 샘플 학습
2번째 batch: 32개 샘플 학습
3번째 batch: 32개 샘플 학습
...
마지막 batch: 남은 샘플 학습
```

------

# 11. 1198개를 32개씩 나누면?

```python
1198 ÷ 32 = 37.4
```

즉:

- 37개 batch는 32개씩
- 마지막 batch는 14개

가 된다.

총:

```python
38개 batch
```

정도가 한 epoch 안에서 학습된다.

------

# 12. batch마다 일어나는 일

각 batch마다:

1. X_train 일부가 RNN에 들어감
2. 예측값 생성
3. y_train과 비교
4. loss 계산
5. backpropagation 수행
6. weight 업데이트

가 일어난다.

------

# 13. 중요한 오해 방지

```python
batch_size = 32
```

는:

```python
32일치 데이터를 본다
```

는 뜻이 아니다.

------

32일치가 아니라:

```python
32개의 sample을 한 번에 처리
```

한다는 뜻이다.

------

# 14. 다시 구분

| 값   | 의미                             |
| ---- | -------------------------------- |
| 60   | 한 sample 안의 과거 60일         |
| 32   | 한 번에 학습하는 sample 개수     |
| 100  | 전체 데이터를 반복 학습하는 횟수 |

------

# 15. 학습 중 loss 확인

학습을 실행하면 화면에:

```python
loss: 0.0531
loss: 0.0022
loss: 0.0015
```

같은 값이 나온다.

------

# 16. loss란?

loss는:

> 예측값과 실제값의 차이

이다.

여기서는 compile 단계에서 설정한:

```python
mean_squared_error
```

를 사용한다.

------

# 17. loss가 줄어든다는 의미

loss가 줄어든다는 것은:

```python
예측값이 실제 주가에 가까워지고 있다
```

는 뜻이다.

------

# 18. 하지만 loss가 너무 작아도 문제

훈련 데이터에서 loss가 너무 작으면:

```python
훈련 데이터만 외운 상태
```

일 수 있다.

이걸:

```python
Overfitting
```

이라고 한다.

------

# 19. 왜 조심해야 하나?

우리가 진짜 원하는 것은:

```python
과거 데이터 잘 맞추기
```

가 아니다.

우리가 원하는 것은:

```python
새로운 test 데이터에서도 추세를 잘 따라가기
```

이다.

------

# 20. 강의에서 말하는 수렴

강의에서는 마지막 10~20 epoch에서 loss가:

```python
0.0015 근처
```

로 크게 줄지 않고 비슷하게 유지된다고 설명한다.

이런 상태를:

```python
convergence
```

즉, 수렴이라고 볼 수 있다.

------

# 21. 현재까지 완성된 전체 코드 흐름

```python
regressor = Sequential()

regressor.add(LSTM(units=50, return_sequences=True, input_shape=(X_train.shape[1], 1)))
regressor.add(Dropout(0.2))

regressor.add(LSTM(units=50, return_sequences=True))
regressor.add(Dropout(0.2))

regressor.add(LSTM(units=50, return_sequences=True))
regressor.add(Dropout(0.2))

regressor.add(LSTM(units=50))
regressor.add(Dropout(0.2))

regressor.add(Dense(units=1))

regressor.compile(
    optimizer='adam',
    loss='mean_squared_error'
)

regressor.fit(
    X_train,
    y_train,
    epochs=100,
    batch_size=32
)
```

------

# 23. Part 2 최종 의미

이제 모델은 단순히 구조만 있는 상태가 아니라:

```python
학습된 RNN 모델
```

이 된다.

즉:

> 과거 60일 주가를 보고 다음 날 주가 추세를 예측할 수 있도록 훈련된 상태

가 된다.

------

# 🎯 최종 핵심 정리

- `fit()`은 실제 학습을 실행하는 함수
- `X_train`은 입력 데이터
- `y_train`은 정답 데이터
- `epochs=100`은 전체 데이터를 100번 반복 학습
- `batch_size=32`는 32개 sample씩 나눠서 학습
- loss는 예측값과 실제값의 차이
- loss가 줄어들면 모델이 학습되고 있다는 뜻
- 하지만 loss를 무조건 0에 가깝게 만드는 것이 목표는 아님
- 너무 낮은 train loss는 overfitting 가능성도 있음

------

# 🔥 한 줄 최종 정리

> 이번 단계는 `regressor.fit(X_train, y_train, epochs=100, batch_size=32)`를 실행하여, 과거 60일 주가 데이터를 입력으로 넣고 다음 날 주가를 맞추도록 RNN을 실제로 학습시키는 과정이다.
