# 📌 Part 2-4: Output Layer 추가하기

이번 단계에서는:

> 마지막 Output Layer(출력층)

를 추가한다.

이제 거의 RNN 구조가 완성되는 단계이다.

현재까지는:

```python
LSTM → LSTM → LSTM → LSTM
```

구조만 만들었고,

아직:

```python
최종 예측값 출력
```

을 하는 층은 없었다.

------

# 1. 이번 단계에서 추가할 코드

## 코드

```python
regressor.add(Dense(units = 1))
```

------

# 2. 왜 Output Layer가 필요한가?

현재 마지막 LSTM은:

```python
시계열 특징(feature)
```

만 추출한 상태이다.

즉:

```python
"주가가 상승하는 패턴 같다"
"변동성이 커지고 있다"
```

같은 내부 특징(hidden state)을 가지고 있을 뿐,

아직:

```python
실제 주가 숫자
```

를 출력하지는 않는다.

------

# 3. 따라서 마지막 변환이 필요

즉:

```python
LSTM 특징 벡터
```

를:

```python
실제 주가 값
```

으로 변환해야 한다.

그 역할을 하는 것이:

```python
Dense Layer
```

이다.

------

# 4. Dense란?

Dense는:

> Fully Connected Layer (완전 연결층)

이다.

------

# 5. Fully Connected 의미

현재 마지막 LSTM의 출력:

```python
50차원 hidden state
```

가 있다고 해보자.

------

Dense layer는:

```python
50개 값을 모두 사용
```

해서:

```python
최종 예측값 1개
```

를 만든다.

------

# 6. 그림 느낌

```python
LSTM Output (50개 특징)
 ↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓
Dense Layer
 ↓
예측 주가 1개
```

------

# 7. 왜 Dense를 사용하는가?

강의에서도 말하듯:

```python
Output Layer는 일반 fully connected layer
```

이기 때문이다.

------

즉:

```python
LSTM
```

은:

```python
시계열 특징 추출기
```

이고,

------

```python
Dense
```

는:

```python
최종 숫자 출력기
```

역할이다.

------

# 8. Dense(units=1)

## 코드

```python
Dense(units = 1)
```

------

# 9. units 의미 복습

Dense에서:

```python
units
```

는:

```python
출력 뉴런 개수
```

이다.

------

# 10. 왜 1개인가?

우리가 예측하는 것은:

```python
다음 날 주가 하나
```

뿐이다.

즉:

```python
T+1 시점의 주가 값
```

하나를 예측한다.

------

# 11. 따라서 출력 차원도 1

즉:

```python
Output Shape = (batch_size, 1)
```

이 된다.

------

# 12. 내부적으로 어떻게 계산될까?

Dense layer는:

```python
y = Wx + b
```

형태 계산을 수행한다.

------

# 13. 입력과 출력

입력:

```python
(batch_size, 50)
```

↓

Dense

↓

출력:

```python
(batch_size, 1)
```

------

# 14. 왜 입력이 (batch_size, 50)인가?

마지막 LSTM은:

```python
return_sequences=False
```

였다.

따라서:

```python
마지막 timestep hidden state만 출력
```

한다.

------

즉 출력 shape는:

```python
(batch_size, 50)
```

이다.

------

# 15. Dense가 최종 주가 생성

Dense layer는:

```python
50차원 특징
```

을 조합해서:

```python
주가 숫자 1개
```

를 생성한다.

------

# 16. 현재까지 완성된 구조

지금까지 만든 전체 구조:

```python
Input
 ↓
LSTM(50)
 ↓
Dropout
 ↓
LSTM(50)
 ↓
Dropout
 ↓
LSTM(50)
 ↓
Dropout
 ↓
LSTM(50)
 ↓
Dropout
 ↓
Dense(1)
```

------

# 19. 왜 Mean Squared Error 사용?

강의에서 말하듯:

이번 문제는:

```python
Regression
```

문제이다.

------

즉:

```python
연속적인 숫자 예측
```

```python
Mean Squared Error(MSE)
```

를 loss로 사용한다.

------

# 20. fit은 무엇인가?

fit은:

> 실제 학습 시작

이다.

------

## 코드 느낌

```python
regressor.fit(X_train, y_train)
```

------

# 21. 왜 X_train을 사용하는가?

강의에서 중요하게 말하는 부분이다.

------

우리는:

```python
training_set
```

이 아니라:

```python
X_train
```

을 사용해야 한다.

------

# 22. 이유

왜냐하면 X_train은:

```python
(1198,60,1)
```

형태의:

```python
RNN 전용 3차원 구조
```

로 이미 변환되어 있기 때문이다.

------

# 23. y_train은 무엇인가?

y_train은:

```python
정답 데이터(ground truth)
```

이다.

------

즉:

```python
과거 60일
```

을 보고,

------

```python
다음 날 실제 주가
```

를 맞추도록 학습한다.

------

# 24. 실제 학습 느낌

```python
입력:
[60일 주가]

정답:
61일째 주가
```

------

LSTM은:

```python
예측값
```

과

```python
실제값(y_train)
```

차이를 줄이는 방향으로 학습한다.

------


  예측


  - fit
