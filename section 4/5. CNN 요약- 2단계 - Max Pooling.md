# 🧠 CNN Step 2: Pooling (Max Pooling)

Convolution과 ReLU를 통해
👉 이미지에서 특징을 추출하고 정리했다.

하지만 아직 문제가 하나 남아 있다.

👉 **이미지의 위치나 형태가 조금만 바뀌어도 제대로 인식할 수 있을까?**

이 문제를 해결하기 위해 사용하는 것이 바로
👉 **Pooling (특히 Max Pooling)**이다.

------

# 1. Pooling이 필요한 이유

같은 대상이라도 이미지에서는 다양한 형태로 나타난다.

예를 들어 치타를 생각해보자.

![4-12](../img\section4\4-12.png)

- 정면을 보는 치타
- 약간 기울어진 치타
- 위치가 다른 치타

👉 모두 같은 “치타”지만
👉 픽셀 기준으로 보면 완전히 다른 이미지이다

------

### ✔ 문제

만약 CNN이
👉 “정확히 같은 위치의 특징”만 찾는다면

- 조금만 위치가 바뀌어도
  → 인식 실패

------

👉 그래서 필요한 개념이

👉 **Spatial Invariance (위치 변화에 강한 성질)**

------

👉 한 줄 정리
→ “위치가 달라도 같은 것으로 인식해야 한다”

------

# 2. Pooling이란 무엇인가?

Pooling은 간단하게 말하면

👉 **특징을 유지하면서 정보를 줄이는 과정**이다

------

특히 Max Pooling은

👉 **작은 영역에서 가장 큰 값만 남긴다**

------

👉 한 줄 정리
→ “중요한 값 하나만 남기고 나머지는 버린다”

------

# 3. Max Pooling 동작 방식

Max Pooling은 다음과 같이 진행된다.

------

## ① 작은 영역 선택 (예: 2×2)

![4-13](../img\section4\4-13.png)

Feature Map에서
작은 영역을 하나 선택한다

------

## ② 최대값 선택

![4-14](../img\section4\4-14.png)

그 영역 안에서
👉 가장 큰 값 하나만 선택

------

## ③ 이동하면서 반복

![4-15](../img\section4\4-15.png)

- 보통 stride = 2 사용
- 겹치지 않게 이동

------

👉 결과

![4-16](../img\section4\4-16.png)

👉 **크기가 줄어든 Feature Map 생성**

------

👉 한 줄 정리
→ “영역별 대표값만 뽑는다”

------

# 4. Pooling의 핵심 효과

Pooling은 단순히 크기만 줄이는 게 아니다.

------

## ✔ ① 특징 유지

큰 값 = 중요한 특징

👉 Max Pooling은 이 값만 남김

→ 특징은 유지됨

------

## ✔ ② 위치 변화에 강해짐

예를 들어

- 특징이 조금 왼쪽에 있든
- 조금 오른쪽에 있든

👉 같은 영역 안에 있으면
👉 동일한 결과가 나옴

------

👉 결과

👉 **위치가 달라도 동일하게 인식 가능**

------

## ✔ ③ 데이터 크기 감소

- 2×2 pooling → 75% 정보 제거
- 계산량 감소
- 속도 증가

------

## ✔ ④ 과적합 방지

불필요한 정보 제거 →

👉 모델이 “쓸데없는 정보까지 외우는 것” 방지

------

👉 한 줄 정리
→ “정보 줄이고, 일반화 성능 올린다”

------

# 5. 다른 Pooling 방식

Max Pooling 외에도 다양한 방식이 있다.

------

### ✔ Mean Pooling (Average Pooling)

- 평균값 사용

------

### ✔ Sum Pooling

- 값을 모두 더함

------

👉 하지만 실제로는

👉 **Max Pooling이 가장 많이 사용된다**

------

# 6. 전체 흐름에서 Pooling의 위치

CNN 흐름은 이렇게 이어진다:

![4-17](../img\section4\4-17.png)

```id="poolflow1"
Convolution → ReLU → Pooling
```

------

👉 의미

- Convolution → 특징 찾기
- ReLU → 특징 정리
- Pooling → 특징 압축

------

👉 한 줄 정리
→ “찾고 → 정리하고 → 줄인다”

------

# 7. Pooling의 진짜 핵심

Pooling의 핵심은 단순한 압축이 아니다.

👉 진짜 목적은 이것이다:

- 특징은 유지하고
- 위치 변화에 강하게 만들고
- 불필요한 정보는 제거한다

------

👉 사람과 비교하면

- 우리는 모든 픽셀을 기억하지 않는다
- 중요한 특징만 기억한다

👉 CNN도 동일하게 동작한다

------

# 8. 핵심 요약

- Pooling은 Feature Map을 줄이는 과정이다
- Max Pooling은 가장 중요한 값만 남긴다
- 위치 변화에 강한 모델을 만든다
- 계산량 감소 + 과적합 방지 효과가 있다

------

# 🎯 한 줄 정리

👉 **“Pooling은 중요한 특징만 남기고 정보를 압축하여, 위치 변화에 강한 모델을 만드는 과정이다.”**
