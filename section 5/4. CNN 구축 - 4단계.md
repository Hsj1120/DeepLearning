# 📌 전체 흐름

```text
모델 설계 → 컴파일 → 학습 → 평가
```

------

# 📌 1. CNN 컴파일 (Compile)

```python
cnn.compile(
    optimizer='adam',
    loss='binary_crossentropy',
    metrics=['accuracy']
)
```

------

## ✔ 컴파일이란?

👉
“모델이 어떻게 학습할지 설정하는 단계”

------

## ✔ 핵심 3가지

### 1️⃣ optimizer = 'adam'

👉 가중치 업데이트 방법

- 오차 줄이기 위한 알고리즘
- 가장 많이 쓰는 기본값

------

### 2️⃣ loss = 'binary_crossentropy'

👉 오차 계산 방법

- 이진 분류 (고양이 vs 개)에 적합

------

### 3️⃣ metrics = ['accuracy']

👉 성능 평가 기준

- 얼마나 맞췄는지

------

## ✔ 한 줄 이해

👉
“어떻게 배우고, 어떻게 틀렸는지 계산하고, 어떻게 평가할지 설정”

------

# 📌 2. CNN 학습 (fit)

```python
cnn.fit(
    x=training_set,
    validation_data=test_set,
    epochs=25
)
```

------

# 📌 ✔ 각 파라미터 의미

------

## ✔ 1️⃣ x = training_set

👉 학습 데이터

- 이미지 증강 적용된 데이터
- 모델이 배우는 대상

------

## ✔ 2️⃣ validation_data = test_set ⭐

👉 평가 데이터

- 학습에 사용하지 않음
- 매 epoch마다 성능 확인

------

## ✔ 핵심 포인트

👉
“학습하면서 동시에 평가한다”

------

## ✔ 3️⃣ epochs = 25

👉 전체 데이터를 몇 번 반복 학습할지

------

## ✔ 예시

- epoch 1 → 전체 데이터 1번 학습
- epoch 25 → 총 25번 반복

------

# 📌 왜 동시에 평가하냐?

👉 CNN (특히 이미지)은 학습이 오래 걸림

그래서:

👉 매 epoch마다 확인

- 성능 올라가는지
- 과적합 발생하는지

------

# 📌 과적합 확인 방법

👉 이런 상황 보면 의심

| 구분           | 값   |
| -------------- | ---- |
| train accuracy | 높음 |
| test accuracy  | 낮음 |

👉 → 과적합 발생

------

# 📌 핵심 흐름 그림

```text
[training_set]
   ↓
CNN 학습 (fit)
   ↓
[validation_data]
   ↓
성능 확인 (accuracy)
```

------

# 📌 중요한 차이 (ANN vs CNN)

| 항목      | ANN     | CNN       |
| --------- | ------- | --------- |
| 평가      | 따로 함 | 동시에 함 |
| 데이터    | 숫자    | 이미지    |
| 학습 속도 | 빠름    | 느림      |

------
