# 📌 CNN 데이터 전처리 핵심 정리

## 1. 시작: 라이브러리 불러오기

CNN을 만들기 위해 필요한 기본 라이브러리

```python
import tensorflow as tf
from keras.preprocessing.image import ImageDataGenerator
```

👉 `ImageDataGenerator`
= 이미지 전처리 + 증강을 담당하는 핵심 도구

------

# 📌 2. 왜 이미지 전처리가 필요한가?

이미지는 그대로 넣으면 안 된다.

👉 이유 2가지:

1. **모델 학습 안정화 (정규화)**
2. **과적합 방지 (데이터 증강)**

------

# 📌 3. 이미지 증강 (🔥 핵심)

## ✔ 왜 하는가?

👉 모델이 특정 이미지에만 맞춰 학습하는 걸 막기 위해

(= 과적합 방지)

------

## ✔ 어떤 걸 하냐?

이미지를 일부 변형해서
👉 “새로운 데이터처럼” 만들어준다

예시:

- 회전
- 확대/축소
- 좌우 반전
- 기울이기

👉 같은 고양이 사진이라도
👉 다양한 형태로 학습됨

------

## ✔ 코드

```python
train_datagen = ImageDataGenerator(
    rescale=1./255,
    shear_range=0.2,
    zoom_range=0.2,
    horizontal_flip=True
)
```

------

## ✔ 각각 의미

- `rescale=1./255`
  👉 픽셀값 0~255 → 0~1 (정규화)
- `shear_range`
  👉 이미지 기울이기
- `zoom_range`
  👉 확대/축소
- `horizontal_flip`
  👉 좌우 반전

------

# 📌 4. 훈련 데이터 연결

```python
training_set = train_datagen.flow_from_directory(
    'dataset/training_set',
    target_size=(64, 64),
    batch_size=32,
    class_mode='binary'
)
```

------

## ✔ 중요한 파라미터

- `target_size=(64, 64)`
  👉 이미지 크기 통일 (속도 ↑)
- `batch_size=32`
  👉 한 번에 32장씩 학습
- `class_mode='binary'`
  👉 고양이 vs 개 (2개 클래스)

------

# 📌 5. 테스트 데이터 전처리 (🔥 중요 차이)

```python
test_datagen = ImageDataGenerator(rescale=1./255)
```

👉 여기 핵심 포인트:

❌ **증강 안 함**
⭕ **정규화만 함**

------

## ✔ 이유

👉 테스트 데이터는 “실제 상황”이라고 가정해야 함

- 변형하면 안 됨
- 원본 그대로 평가해야 함

------

# 📌 6. 테스트 데이터 연결

```python
test_set = test_datagen.flow_from_directory(
    'dataset/test_set',
    target_size=(64, 64),
    batch_size=32,
    class_mode='binary'
)
```

------

# 📌 7. 전체 흐름 정리

👉 훈련 데이터
= 정규화 + 증강

👉 테스트 데이터
= 정규화만

------

# 🔥 핵심 요약 (진짜 중요)

👉 이미지 증강은 **훈련 데이터에만 적용**

👉 테스트 데이터는
→ 절대 건드리지 않는다

------
